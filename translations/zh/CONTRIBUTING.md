# 贡献

感谢您对贡献的兴趣！我们欢迎对本课程的拉取请求和改进。提交更改时，请遵循以下指南：

1. **提交中不要包含秘密信息。**切勿提交 API 密钥或任何机密材料。请使用本地 `.env` 进行个人测试。
2. **保持更改聚焦。**小而有针对性的拉取请求更易于审查和维护。如果您修复多个问题，请分别提交。
3. **尽可能添加测试。**单元测试或集成测试有助于捕捉回归。对于 Spring Boot 模块，请在 `src/test/java` 文件夹下包含测试。
4. **遵循代码风格约定。**使用现有项目结构作为指导。我们依赖 Maven 管理依赖，并使用 Spring Boot Maven 插件打包应用程序。

感谢您帮助改进本课程！
