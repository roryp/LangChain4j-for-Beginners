# Bidrag

Tak for din interesse i at bidrage! Vi byder pull requests og forbedringer til dette kursus velkommen. Følg venligst disse retningslinjer, når du indsender ændringer:

1. **Ingen hemmeligheder i commits.** Indsæt aldrig API-nøgler eller andet fortroligt materiale. Brug en lokal `.env` til din egen testning.
2. **Hold ændringer fokuserede.** Små, afgrænsede pull requests er nemmere at gennemgå og vedligeholde. Hvis du retter flere problemer, så indsend dem venligst separat.
3. **Tilføj tests hvor muligt.** Unit- eller integrationstests hjælper med at fange regressionsfejl. For Spring Boot-moduler, inkluder tests under mappen `src/test/java`.
4. **Følg kode-stil konventioner.** Brug den eksisterende projektstruktur som vejledning. Vi bruger Maven til at håndtere afhængigheder og Spring Boot Maven-plugin til at pakke applikationer.

Tak fordi du hjælper med at gøre dette kursus bedre!
