# Contribuire

Grazie per il tuo interesse a contribuire! Accogliamo con piacere pull request e miglioramenti a questo corso. Si prega di seguire queste linee guida quando si inviano modifiche:

1. **Niente segreti nei commit.** Non inserire mai chiavi API o materiale riservato. Usa un file `.env` locale per i tuoi test.
2. **Mantieni le modifiche mirate.** Pull request piccole e focalizzate sono più facili da revisionare e mantenere. Se stai correggendo più problemi, inviali separatamente.
3. **Aggiungi test quando possibile.** Test unitari o di integrazione aiutano a individuare regressioni. Per i moduli Spring Boot, includi i test nella cartella `src/test/java`.
4. **Segui le convenzioni di stile del codice.** Usa la struttura esistente del progetto come guida. Facciamo affidamento su Maven per gestire le dipendenze e sul plugin Spring Boot Maven per impacchettare le applicazioni.

Grazie per aver contribuito a migliorare questo corso!
