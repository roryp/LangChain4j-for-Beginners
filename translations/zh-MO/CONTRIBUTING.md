# Contributing

多謝你有興趣作出貢獻！我哋歡迎拉取請求同改進呢個課程。提交更改時，請遵守以下指引：

1. **提交時唔好有秘密資料。** 千祈唔好提交 API 金鑰或者任何機密資料。測試時請用本地 `.env` 文件。
2. **保持更改集中。** 細小、範圍明確嘅拉取請求較易審核同維護。如果你修正多個問題，請分開提交。
3. **盡量加測試。** 單元測試或者整合測試有助捕捉回歸問題。對於 Spring Boot 模組，請喺 `src/test/java` 文件夾下加入測試。
4. **遵守代碼風格慣例。** 以現有項目結構作指引。我哋用 Maven 管理依賴，用 Spring Boot Maven 插件打包應用程式。

多謝你幫助令呢個課程更好！
