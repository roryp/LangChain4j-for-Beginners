## मुख्य संकल्पना

**AI एजंट** - AI वापरून स्वायत्तपणे विचार करणारी आणि क्रिया करणारी प्रणाली. [मॉड्यूल 04](../04-tools/README.md)

**चेन** - असंख्य क्रियांची शृंखला जिथे आउटपुट पुढील टप्प्यात वापरले जाते.

**चंकिंग** - दस्तऐवज लहान तुकड्यांमध्ये विभागणे. सामान्यतः: 300-500 टोकन्स ओव्हरलॅपसह. [मॉड्यूल 03](../03-rag/README.md)

**कॉन्टेक्स्ट विंडो** - मॉडेल प्रक्रिया करू शकणारे जास्तीत जास्त टोकन्स. GPT-5: 400K टोकन्स.

**एम्बेडिंग्ज** - मजकुराचा अर्थ दर्शविणारे संख्यात्मक वेक्टर. [मॉड्यूल 03](../03-rag/README.md)

**फंक्शन कॉलिंग** - मॉडेल बाह्य फंक्शन्स कॉल करण्यासाठी संरचित विनंत्या तयार करते. [मॉड्यूल 04](../04-tools/README.md)

**हॅलुसिनेशन** - जेव्हा मॉडेल चुकीची पण संभाव्य माहिती तयार करते.

**प्रांप्ट** - भाषा मॉडेलसाठी मजकूर इनपुट. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**समानार्थी शोध** - कीवर्ड न वापरता अर्थाने शोधणे, एम्बेडिंग्ज वापरून. [मॉड्यूल 03](../03-rag/README.md)

**स्टेटफुल विरुद्ध स्टेटलेस** - स्टेटलेस: कोणतीही मेमरी नाही. स्टेटफुल: संभाषणाचा इतिहास ठेवतो. [मॉड्यूल 01](../01-introduction/README.md)

**टोकन्स** - मूलभूत मजकूर एकक जे मॉडेल प्रक्रिया करतात. खर्च आणि मर्यादांवर प्रभाव टाकते. [मॉड्यूल 01](../01-introduction/README.md)

**टूल चेनिंग** - साधनांची सलग अंमलबजावणी जिथे आउटपुट पुढच्या कॉलची माहिती देते. [मॉड्यूल 04](../04-tools/README.md)

## LangChain4j घटक

**AiServices** - प्रकार-सुरक्षित AI सेवा इंटरफेस तयार करते.

**OpenAiOfficialChatModel** - OpenAI आणि Azure OpenAI मॉडेलसाठी एकसंध क्लायंट.

**OpenAiOfficialEmbeddingModel** - OpenAI अधिकृत क्लायंट वापरून एम्बेडिंग्ज तयार करते (OpenAI आणि Azure OpenAI दोन्हीसाठी समर्थित).

**ChatModel** - भाषा मॉडेलसाठी मुख्य इंटरफेस.

**ChatMemory** - संभाषणाचा इतिहास राखते.

**ContentRetriever** - RAG साठी संबंधित दस्तऐवज चंक शोधते.

**DocumentSplitter** - दस्तऐवज चंकमध्ये विभागतो.

**EmbeddingModel** - मजकूर संख्यात्मक वेक्टरमध्ये रूपांतरित करते.

**EmbeddingStore** - एम्बेडिंग्ज संग्रहित आणि पुनर्प्राप्त करते.

**MessageWindowChatMemory** - अलीकडील संदेशांची स्लायडिंग विंडो राखते.

**PromptTemplate** - `{{variable}}` प्लेसहोल्डर्ससह पुनर्वापरयोग्य प्रांप्ट तयार करते.

**TextSegment** - मेटाडेटासह मजकूराचा तुकडा. RAG मध्ये वापरला जातो.

**ToolExecutionRequest** - टूल अंमलबजावणी विनंतीचे प्रतिनिधित्व करते.

**UserMessage / AiMessage / SystemMessage** - संभाषण संदेश प्रकार.

## AI/ML संकल्पना

**फ्यू-शॉट लर्निंग** - प्रांप्टमध्ये उदाहरणे देणे. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**लार्ज लँग्वेज मॉडेल (LLM)** - प्रचंड मजकूर डेटावर प्रशिक्षित AI मॉडेल्स.

**रिझनिंग एफर्ट** - GPT-5 पॅरामीटर जो विचारांची खोलाई नियंत्रित करतो. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**टेम्परेचर** - आउटपुटचे अराजकपण नियंत्रित करते. कमी = निश्चित, जास्त = सर्जनशील.

**व्हेक्टर डेटाबेस** - एम्बेडिंग्जसाठी विशेष डेटाबेस. [मॉड्यूल 03](../03-rag/README.md)

**झिरो-शॉट लर्निंग** - उदाहरणांशिवाय कामे करणे. [मॉड्यूल 02](../02-prompt-engineering/README.md)

## गार्डरेल्स - [मॉड्यूल 00](../00-quick-start/README.md)

**डिफेन्स इन डेप्थ** - 多स्तरीय सुरक्षा दृष्टीकोन जो ऍप्लिकेशन-स्तरीय गार्डरेल्स आणि प्रदाता सुरक्षितता फिल्टर्स एकत्र करतो.

**हार्ड ब्लॉक** - प्रदाता गंभीर सामग्री उल्लंघनांसाठी HTTP 400 त्रुटी फेकतो.

**InputGuardrail** - LLM पर्यंत पोहोचण्यापूर्वी वापरकर्ता इनपुट वैधतेसाठी LangChain4j इंटरफेस. हानीकारक प्रांप्ट्स लवकर अडवून खर्च आणि विलंब कमी करतो.

**InputGuardrailResult** - गार्डरेल वैधतेसाठी परतावा प्रकार: `success()` किंवा `fatal("reason")`.

**OutputGuardrail** - AI प्रतिसाद वापरकर्त्यांना परत करण्यापूर्वी वैधतेसाठी इंटरफेस.

**Provider Safety Filters** - AI प्रदात्यांकडून (उदा. GitHub Models) API स्तरावर उल्लंघने पकडणारे अंगभूत सामग्री फिल्टर्स.

**Soft Refusal** - मॉडेल विनम्रपणे उत्तर देण्यास नकार देते, त्रुटी न फेकता.

## प्रांप्ट अभियांत्रिकी - [मॉड्यूल 02](../02-prompt-engineering/README.md)

**चेन-ऑफ-थॉट** - अधिक अचूकतेसाठी टप्प्याटप्प्याने विचार.

**बाधित आउटपुट** - विशिष्ट स्वरूप किंवा संरचना बळजबरीने लागू करणे.

**उच्च उत्सुकता** - GPT-5 साठी काळजीपूर्वक विचार करण्याचा पॅटर्न.

**कमी उत्सुकता** - GPT-5 साठी तत्पर उत्तर देण्याचा पॅटर्न.

**मल्टी-टर्न संभाषण** - देवाणघेवाणीत संदर्भ राखणे.

**भूमिका-आधारित प्रांप्टिंग** - सिस्टम संदेशांद्वारे मॉडेल व्यक्तिमत्व सेट करणे.

**स्व-परावर्तन** - मॉडेल आपले आउटपुट मूल्यांकन आणि सुधारते.

**संरचित विश्लेषण** - निश्चित मूल्यांकन चौकट.

**कार्य अंमलबजावणी पॅटर्न** - योजना → अंमलबजावणी → सारांश.

## RAG (रिट्रीवल-अगुमेंटेड जनरेशन) - [मॉड्यूल 03](../03-rag/README.md)

**दस्तऐवज प्रक्रिया पाइपलाइन** - लोड → चंक → एम्बेड → स्टोर.

**इन-मेमरी एम्बेडिंग स्टोर** - चाचणीसाठी तात्पुरते संग्रह.

**RAG** - प्रतिक्रिया स्थानिक करण्यासाठी पुनर्प्राप्ती व निर्मिती एकत्र करतो.

**समानता गुणांकन** - सेमॅंटिक समानतेचा मोजमाप (0-1).

**स्त्रोत संदर्भ** - पुनर्प्राप्त सामग्री बद्दल मेटाडेटा.

## एजंट आणि साधने - [मॉड्यूल 04](../04-tools/README.md)

**@Tool Annotation** - Java पद्धतींना AI-कॉल करण्यायोग्य साधने म्हणून चिन्हांकित करते.

**ReAct पॅटर्न** - विचार करा → क्रिया करा → निरीक्षण करा → पुनरावृत्ती करा.

**सेशन व्यवस्थापन** - वेगवेगळ्या वापरकर्त्यांसाठी स्वतंत्र संदर्भ.

**टूल** - AI एजंट कॉल करू शकणारा फंक्शन.

**टूल वर्णन** - साधनाचा हेतू आणि पॅरामिटर्सची माहिती.

## एजंटिक मॉड्यूल - [मॉड्यूल 05](../05-mcp/README.md)

**@Agent Annotation** - इंटरफेसेसना AI एजंट म्हणून चिन्हांकित करतो ज्यात घोषणात्मक वर्तन परिभाषा असते.

**Agent Listener** - `beforeAgentInvocation()` आणि `afterAgentInvocation()` द्वारे एजंट अंमलबजावणीचे निरीक्षण करण्यासाठी हुक.

**Agentic Scope** - शेअर केलेली मेमरी जिथे एजंट आउटपुट `outputKey` वापरून साठवतात आणि पुढील एजंटसाठी वापरतात.

**AgenticServices** - `agentBuilder()` आणि `supervisorBuilder()` वापरून एजंट तयार करण्यासाठी फॅक्टरी.

**Conditional Workflow** - परिस्थितीवर आधारित भिन्न विशेषज्ञ एजंटांकडे मार्गदर्शन करणे.

**Human-in-the-Loop** - मानवी तपासणी किंवा मान्यता जोडणारा वर्कफ्लो पॅटर्न.

**langchain4j-agentic** - घोषणात्मक एजंट बिल्डिंगसाठी Maven अवलंबित्व (प्रयोगात्मक).

**Loop Workflow** - एक अट पूर्ण होईपर्यंत एजंटची पुन्हा पुन्हा अंमलबजावणी (उदा., गुणवत्ता गुणांकन ≥ 0.8).

**outputKey** - एजंट अॅनोटेशन पॅरामीटर, जे Agentic Scope मध्ये निकाल कुठे साठवायचा ते ठरवते.

**Parallel Workflow** - स्वतंत्र कामांसाठी एकाच वेळी अनेक एजंट चालवणे.

**Response Strategy** - सुपरवायझर अंतिम उत्तर कसे मांडतो: LAST, SUMMARY, किंवा SCORED.

**Sequential Workflow** - एजंट्सचे आदेशाने कार्यवाही जिथे आउटपुट पुढच्या टप्प्यात जाते.

**Supervisor Agent Pattern** - प्रगत एजंटिक पॅटर्न जिथे सुपरवायझर LLM डायनॅमिक पद्धतीने कोणते सब-एजंट कॉल करायचे ते ठरवतो.

## मॉडेल कॉन्टेक्स्ट प्रोटोकॉल (MCP) - [मॉड्यूल 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j मध्ये MCP एकत्रिकरणासाठी Maven अवलंबित्व.

**MCP** - मॉडेल कॉन्टेक्स्ट प्रोटोकॉल: बाह्य साधनांसह AI अनुप्रयोग जोडण्याचा मानक. एकदा तयार करा, सर्वत्र वापरा.

**MCP क्लायंट** - MCP सर्व्हरशी जोडणारे आणि साधने वापरणारे अनुप्रयोग.

**MCP सर्व्हर** - साफ वर्णन आणि पॅरामीटर स्कीमासह MCP द्वारे साधने प्रकट करणारी सेवा.

**McpToolProvider** - MCP साधने AI सेवा आणि एजंटसाठी वापरण्यासाठी झाकणारी LangChain4j घटक.

**McpTransport** - MCP संवादासाठी इंटरफेस. अंमलबजावण्या मध्ये Stdio आणि HTTP समाविष्ट.

**Stdio ट्रान्सपोर्ट** - stdin/stdout द्वारे स्थानिक प्रक्रिया ट्रान्सपोर्ट. फाईलसिस्टम ऍक्सेस किंवा कमांड-लाइन साधनांसाठी उपयुक्त.

**StdioMcpTransport** - LangChain4j अंमलबजावणी जी MCP सर्व्हर उपप्रक्रिया म्हणून सुरू करते.

**टूल डिस्कव्हरी** - क्लायंट उपलब्ध साधने, वर्णने आणि स्कीमासाठी सर्व्हरला क्वेरी करतो.
