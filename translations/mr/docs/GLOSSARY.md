## Core Concepts

**एआय एजंट** - अशी प्रणाली जी स्वायत्तपणे विचार करण्यासाठी आणि कृती करण्यासाठी एआय वापरते. [मॉड्यूल 04](../04-tools/README.md)

**चेन** - क्रियांची अशी मालिका ज्यात एका टप्प्याचा आउटपुट पुढच्या टप्प्यात दिला जातो.

**चंकिंग** - दस्तऐवजांना लहान तुकड्यांमध्ये विभाजित करणे. सामान्यतः: 300-500 टोकन्स ओव्हरलॅपसह. [मॉड्यूल 03](../03-rag/README.md)

**संदर्भ विंडो** - मॉडेल ज्या कमाल टोकन्स प्रक्रिया करू शकते. GPT-5: 400K टोकन्स.

**एम्बेडिंग्ज** - मजकुराच्या अर्थाचे प्रतिनिधित्व करणारे संख्यात्मक वेक्टर. [मॉड्यूल 03](../03-rag/README.md)

**फंक्शन कॉलिंग** - मॉडेल बाह्य फंक्शन्सना कॉल करण्यासाठी संरचित विनंत्या तयार करते. [मॉड्यूल 04](../04-tools/README.md)

**हॅल्युसिनेशन** - जेव्हा मॉडेल चुकीची परंतु संभाव्य वाटणारी माहिती तयार करतात.

**प्रॉम्प्ट** - भाषा मॉडेलसाठी मजकूर इनपुट. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**सिमॅंटिक शोध** - कीवर्ड ऐवजी एम्बेडिंग्ज वापरून अर्थानुसार शोध. [मॉड्यूल 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: स्मृती नाही. Stateful: संभाषण इतिहास राखतो. [मॉड्यूल 01](../01-introduction/README.md)

**टोकन्स** - मॉडेल्स जे मूलभूत मजकूर एकके प्रक्रिया करतात. खर्च आणि मर्यादा यावर परिणाम होतो. [मॉड्यूल 01](../01-introduction/README.md)

**टूल चेनिंग** - अशी साधनांची क्रमवार अंमलबजावणी जिथे एका साधनाचा आउटपुट पुढच्या कॉलला माहिती पुरवतो. [मॉड्यूल 04](../04-tools/README.md)

## LangChain4j Components

**AiServices** - टाइप-सुरक्षित AI सेवा इंटरफेस तयार करते.

**OpenAiOfficialChatModel** - OpenAI आणि Azure OpenAI मॉडेल्ससाठी एकत्रित क्लायंट.

**OpenAiOfficialEmbeddingModel** - OpenAI Official क्लायंट वापरून एम्बेडिंग्ज तयार करते (OpenAI आणि Azure OpenAI दोन्हीना सपोर्ट करते).

**ChatModel** - भाषा मॉडेल्ससाठी मुख्य इंटरफेस.

**ChatMemory** - संभाषणाचा इतिहास जतन करते.

**ContentRetriever** - RAG साठी संबंधित दस्तऐवज तुकडे शोधतो.

**DocumentSplitter** - दस्तऐवजांना तुकड्यांमध्ये विभाजित करते.

**EmbeddingModel** - मजकूराला संख्यात्मक वेक्टरमध्ये रूपांतरित करते.

**EmbeddingStore** - एम्बेडिंग्ज साठवते आणि पुनर्प्राप्त करते.

**MessageWindowChatMemory** - अलीकडील संदेशांचे स्लाइडिंग विंडो जतन ठेवते.

**PromptTemplate** - `{{variable}}` होल्डर सहित पुनर्वापरयोग्य प्रॉम्प्ट्स तयार करते.

**TextSegment** - मेटाडेटासह मजकूर तुकडा. RAG मध्ये वापरला जातो.

**ToolExecutionRequest** - साधन अंमलबजावणीची विनंती दर्शवतो.

**UserMessage / AiMessage / SystemMessage** - संभाषण संदेश प्रकार.

## AI/ML Concepts

**कमी-शॉट शिक्षण** - प्रॉम्प्ट्समध्ये उदाहरणे देणे. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**मोठे भाषा मॉडेल (LLM)** - विशाल मजकूर डेटावर प्रशिक्षित AI मॉडेल्स.

**तर्कशक्तीचा प्रयत्न** - GPT-5 पॅरामीटर जो विचार करण्याच्या खोलीचे नियंत्रण करतो. [मॉड्यूल 02](../02-prompt-engineering/README.md)

**तापमान** - आउटपुटची अनियम्यता नियंत्रित करते. कमी = ठराविक, जास्त = सर्जनशील.

**व्हेक्टर डेटाबेस** - एम्बेडिंग्जसाठी विशेषीकृत डेटाबेस. [मॉड्यूल 03](../03-rag/README.md)

**झिरो-शॉट शिक्षण** - उदाहरणांशिवाय काम करणे. [मॉड्यूल 02](../02-prompt-engineering/README.md)

## प्रॉम्प्ट अभियांत्रिकी - [मॉड्यूल 02](../02-prompt-engineering/README.md)

**चेन-ऑफ-थॉट** - अधिक अचूकतेसाठी पायरी-दर-पायरी तर्कप्रक्रिया.

**बाध्य केलेले आउटपुट** - विशिष्ट फॉरमॅट किंवा संरचना लागू करणे.

**उच्च उत्सुकता** - सखोल तर्कासाठी GPT-5 पॅटर्न.

**कमी उत्सुकता** - जलद उत्तरे देण्यासाठी GPT-5 पॅटर्न.

**बहु-फेरांचे संभाषण** - अदलाबदल दरम्यान संदर्भ राखणे.

**भूमिका-आधारित प्रॉम्प्टिंग** - सिस्टम संदेशांद्वारे मॉडेलला व्यक्तिमत्व सेट करणे.

**स्व-प्रतिबिंब** - मॉडेल त्याचे आउटपुट मूल्यांकन करते आणि सुधारते.

**रचित विश्लेषण** - निश्चित मूल्यांकन ढांचा.

**कार्य अंमलबजावणी नमुना** - योजना → अंमल → सारांश.

## RAG (रिट्रीव्हल-ऑगमेंटेड जनरेशन) - [मॉड्यूल 03](../03-rag/README.md)

**दस्तऐवज प्रक्रिया पाईपलाइन** - Load → chunk → embed → store.

**इन-मेमरी एम्बेडिंग स्टोर** - चाचणीसाठी अस्थायी स्टोरेज.

**RAG** - प्रतिसादांना ग्राउंड करण्यासाठी पुनर्प्राप्ती आणि जनरेशन यांचे संयोजन.

**समानता स्कोर** - सिमॅंटिक समानतेचे मापन (0-1).

**स्रोत संदर्भ** - पुनर्प्राप्त केलेल्या सामग्रीबद्दल मेटाडेटा.

## एजंट आणि साधने - [मॉड्यूल 04](../04-tools/README.md)

**@Tool Annotation** - Java मेथड्सना AI-कॉल करण्यायोग्य साधने म्हणून चिन्हांकित करते.

**ReAct Pattern** - Reason → Act → Observe → Repeat.

**सत्र व्यवस्थापन** - वेगवेगळ्या वापरकर्त्यांसाठी स्वतंत्र संदर्भ.

**Tool** - एआय एजंट कॉल करू शकणारे फंक्शन.

**Tool Description** - साधनाच्या उद्देश आणि पॅरामीटर्सचे दस्तऐवजीकरण.

## मॉडेल संदर्भ प्रोटोकॉल (MCP) - [मॉड्यूल 05](../05-mcp/README.md)

**MCP** - AI अ‍ॅप्सना बाह्य साधनांशी जोडण्यासाठी मानक.

**MCP Client** - MCP सर्व्हर्सशी कनेक्ट होणारे अ‍ॅप्लिकेशन.

**MCP Server** - MCP द्वारे साधने उपलब्ध करणारी सेवा.

**Stdio Transport** - stdin/stdout मार्फत सर्व्हर उपप्रक्रिया म्हणून.

**Tool Discovery** - क्लायंट उपलब्ध साधनांसाठी सर्व्हरला क्वेरी करतो.

## Azure सेवा - [मॉड्यूल 01](../01-introduction/README.md)

**Azure AI Search** - व्हेक्टर क्षमतांसह क्लाऊड शोध. [मॉड्यूल 03](../03-rag/README.md)

**Azure Developer CLI (azd)** - Azure संसाधने डिप्लॉय करते.

**Azure OpenAI** - मायक्रोसॉफ्टची एंटरप्राइझ एआय सेवा.

**Bicep** - Azure इन्फ्रास्ट्रक्चर-ऐस-कोड भाषा. [इन्फ्रास्ट्रकچر मार्गदर्शक](../01-introduction/infra/README.md)

**Deployment Name** - Azure मध्ये मॉडेल डिप्लॉयमेंटसाठी नाव.

**GPT-5** - तर्क नियंत्रणासह नवीनतम OpenAI मॉडेल. [मॉड्यूल 02](../02-prompt-engineering/README.md)
