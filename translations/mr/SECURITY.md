## सुरक्षा

मायक्रोसॉफ्ट आपल्या सॉफ्टवेअर उत्पादनांची आणि सेवा सुरक्षा गांभीर्याने घेतो, ज्यामध्ये आमच्या GitHub संस्थांद्वारे व्यवस्थापित सर्व स्रोत कोड रिपॉझिटरीजचा समावेश आहे, ज्यात [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) आणि [Xamarin](https://github.com/xamarin) यांचा समावेश आहे.

आपणास असे वाटत असल्यास की आपण कोणत्याही मायक्रोसॉफ्ट-स्वामित्वाच्या रिपॉझिटरीमध्ये [मायक्रोसॉफ्टच्या सुरक्षा त्रुटीच्या व्याख्येनुसार](https://aka.ms/security.md/definition) सुरक्षा त्रुटी सापडली आहे, तर कृपया खाली दिल्याप्रमाणे आम्हाला त्याची माहिती द्या.

## सुरक्षा समस्या नोंदविणे

**कृपया सार्वजनिक GitHub समस्यांद्वारे सुरक्षा त्रुटी नोंदवू नका.**

त्याऐवजी, कृपया मायक्रोसॉफ्ट सुरक्षा प्रतिसाद केंद्र (MSRC) येथे [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) वर त्यांची नोंद करा.

आपण लॉगिन न करता सबमिट करायचे असल्यास, [secure@microsoft.com](mailto:secure@microsoft.com) या ईमेलवर पाठवा. शक्य असल्यास, आमच्या PGP कीने आपला संदेश एन्क्रिप्ट करा; कृपया ते [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) वरून डाउनलोड करा.

आपण २४ तासांच्या आत प्रतिसाद मिळवू शकता. काही कारणास्तव जर प्रतिसाद न मिळाला, तर कृपया ईमेलद्वारे फॉलो-अप करा जेणेकरून आम्हाला आपला मूळ संदेश प्राप्त झाला आहे याची खात्री होईल. अधिक माहिती [microsoft.com/msrc](https://www.microsoft.com/msrc) येथे उपलब्ध आहे.

कृपया खालील विनंती केलेली माहिती (जितकी शक्य असेल तितकी) समाविष्ट करा जेणेकरून आम्हाला संभाव्य समस्येचा प्रकार आणि व्याप्ती अधिक चांगल्या प्रकारे समजेल:

  * समस्येचा प्रकार (उदा. बफर ओव्हरफ्लो, SQL इंजेक्शन, क्रॉस-साइट स्क्रिप्टिंग, इत्यादी)
  * समस्येच्या प्रकटीकरणाशी संबंधित स्रोत फाइल(चा) पूर्ण मार्ग
  * प्रभावित स्रोत कोडचे स्थान (टॅग/ब्रांच/कमिट किंवा थेट URL)
  * समस्या पुनरुत्पादित करण्यासाठी आवश्यक कोणतीही विशेष संरचना
  * समस्या पुनरुत्पादित करण्यासाठी टप्प्याटप्प्याने सूचना
  * पुरावा-आधारित संकल्पना किंवा शोषण कोड (जर शक्य असेल तर)
  * समस्येचा परिणाम, ज्यामध्ये हल्लेखोर कसा या समस्येचा फायदा घेऊ शकतो याचा समावेश आहे

ही माहिती आम्हाला आपली तक्रार अधिक जलद तपासण्यास मदत करेल.

जर आपण बग बाउंटीसाठी तक्रार करत असाल, तर अधिक पूर्ण अहवाल उच्च बाउंटी पुरस्कारासाठी योगदान देऊ शकतात. कृपया आमच्या [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) पृष्ठाला भेट द्या आमच्या सक्रिय कार्यक्रमांबद्दल अधिक तपशीलांसाठी.
