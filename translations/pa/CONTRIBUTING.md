# Contributing

ਤੁਹਾਡੇ ਯੋਗਦਾਨ ਵਿੱਚ ਦਿਲਚਸਪੀ ਲਈ ਧੰਨਵਾਦ! ਅਸੀਂ ਇਸ ਕੋਰਸ ਵਿੱਚ ਪੂਲ ਰਿਕਵੇਸਟਾਂ ਅਤੇ ਸੁਧਾਰਾਂ ਦਾ ਸਵਾਗਤ ਕਰਦੇ ਹਾਂ। ਕਿਰਪਾ ਕਰਕੇ ਬਦਲਾਅ ਸੌਂਪਣ ਸਮੇਂ ਇਹਨਾਂ ਨਿਯਮਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ:

1. **ਕਮਿਟਾਂ ਵਿੱਚ ਕੋਈ ਰਾਜ਼ ਨਾ ਰੱਖੋ।** ਕਦੇ ਵੀ API ਕੁੰਜੀਆਂ ਜਾਂ ਕੋਈ ਗੁਪਤ ਸਮੱਗਰੀ ਕਮਿਟ ਨਾ ਕਰੋ। ਆਪਣੇ ਟੈਸਟਿੰਗ ਲਈ ਸਥਾਨਕ `.env` ਵਰਤੋਂ।
2. **ਬਦਲਾਅ ਨੂੰ ਕੇਂਦਰਿਤ ਰੱਖੋ।** ਛੋਟੇ, ਨਿਰਧਾਰਿਤ ਪੂਲ ਰਿਕਵੇਸਟਾਂ ਦੀ ਸਮੀਖਿਆ ਅਤੇ ਸੰਭਾਲ ਆਸਾਨ ਹੁੰਦੀ ਹੈ। ਜੇ ਤੁਸੀਂ ਕਈ ਸਮੱਸਿਆਵਾਂ ਠੀਕ ਕਰ ਰਹੇ ਹੋ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਉਨ੍ਹਾਂ ਨੂੰ ਵੱਖ-ਵੱਖ ਸੌਂਪੋ।
3. **ਜਿੱਥੇ ਸੰਭਵ ਹੋਵੇ, ਟੈਸਟ ਸ਼ਾਮਲ ਕਰੋ।** ਯੂਨਿਟ ਜਾਂ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਟੈਸਟ ਰਿਗ੍ਰੈਸ਼ਨ ਫੜਨ ਵਿੱਚ ਮਦਦ ਕਰਦੇ ਹਨ। ਸਪ੍ਰਿੰਗ ਬੂਟ ਮੋਡੀਊਲਾਂ ਲਈ, `src/test/java` ਫੋਲਡਰ ਹੇਠਾਂ ਟੈਸਟ ਸ਼ਾਮਲ ਕਰੋ।
4. **ਕੋਡ ਸਟਾਈਲ ਰਿਵਾਜਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ।** ਮੌਜੂਦਾ ਪ੍ਰੋਜੈਕਟ ਢਾਂਚੇ ਨੂੰ ਮਾਰਗਦਰਸ਼ਨ ਵਜੋਂ ਵਰਤੋਂ। ਅਸੀਂ Maven 'ਤੇ ਨਿਰਭਰ ਕਰਦੇ ਹਾਂ ਡਿਪੈਂਡੈਂਸੀਜ਼ ਨੂੰ ਸੰਭਾਲਣ ਲਈ ਅਤੇ ਸਪ੍ਰਿੰਗ ਬੂਟ Maven ਪਲੱਗਇਨ ਨੂੰ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਪੈਕੇਜ ਕਰਨ ਲਈ।

ਇਸ ਕੋਰਸ ਨੂੰ ਬਿਹਤਰ ਬਣਾਉਣ ਵਿੱਚ ਸਹਾਇਤਾ ਕਰਨ ਲਈ ਧੰਨਵਾਦ!
