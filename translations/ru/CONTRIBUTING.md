# Contributing

Спасибо за ваш интерес к участию! Мы приветствуем pull-запросы и улучшения этого курса. Пожалуйста, следуйте этим рекомендациям при отправке изменений:

1. **Никаких секретов в коммитах.** Никогда не коммитьте API-ключи или любые конфиденциальные материалы. Используйте локальный `.env` для собственного тестирования.
2. **Сохраняйте фокус изменений.** Небольшие, ограниченные pull-запросы легче просматривать и поддерживать. Если вы исправляете несколько проблем, пожалуйста, отправляйте их отдельно.
3. **Добавляйте тесты, где возможно.** Юнит- или интеграционные тесты помогают обнаруживать регрессии. Для модулей Spring Boot включайте тесты в папку `src/test/java`.
4. **Следуйте соглашениям по стилю кода.** Используйте существующую структуру проекта как руководство. Мы используем Maven для управления зависимостями и плагин Spring Boot Maven для упаковки приложений.

Спасибо, что помогаете сделать этот курс лучше!
