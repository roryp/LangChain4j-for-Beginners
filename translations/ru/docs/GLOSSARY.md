## Core Concepts

**AI Agent** - Система, использующая ИИ для рассуждений и автономных действий. [Модуль 04](../04-tools/README.md)

**Chain** - Последовательность операций, где выход одной служит входом для следующего шага.

**Chunking** - Разбиение документов на более мелкие части. Типично: 300-500 токенов с перекрытием. [Модуль 03](../03-rag/README.md)

**Context Window** - Максимальное количество токенов, которое модель может обработать. GPT-5: 400K токенов.

**Embeddings** - Числовые векторы, представляющие смысл текста. [Модуль 03](../03-rag/README.md)

**Function Calling** - Модель генерирует структурированные запросы для вызова внешних функций. [Модуль 04](../04-tools/README.md)

**Hallucination** - Когда модели генерируют неверную, но правдоподобную информацию.

**Prompt** - Текстовый ввод в языковую модель. [Модуль 02](../02-prompt-engineering/README.md)

**Semantic Search** - Поиск по смыслу с использованием эмбеддингов, а не по ключевым словам. [Модуль 03](../03-rag/README.md)

**Stateful vs Stateless** - Без состояния: нет памяти. С сохранением состояния: поддерживает историю беседы. [Модуль 01](../01-introduction/README.md)

**Tokens** - Базовые текстовые единицы, которые модели обрабатывают. Влияют на стоимость и ограничения. [Модуль 01](../01-introduction/README.md)

**Tool Chaining** - Последовательное выполнение инструментов, где выход используется для следующего вызова. [Модуль 04](../04-tools/README.md)

## LangChain4j Components

**AiServices** - Создаёт типобезопасные интерфейсы AI-сервисов.

**OpenAiOfficialChatModel** - Унифицированный клиент для моделей OpenAI и Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - Создаёт эмбеддинги с использованием официального клиента OpenAI (поддерживает OpenAI и Azure OpenAI).

**ChatModel** - Основной интерфейс для языковых моделей.

**ChatMemory** - Поддерживает историю беседы.

**ContentRetriever** - Находит релевантные фрагменты документов для RAG.

**DocumentSplitter** - Разбивает документы на фрагменты.

**EmbeddingModel** - Преобразует текст в числовые векторы.

**EmbeddingStore** - Сохраняет и извлекает эмбеддинги.

**MessageWindowChatMemory** - Поддерживает скользящее окно последних сообщений.

**PromptTemplate** - Создаёт повторно используемые подсказки с `{{variable}}` заполнителями.

**TextSegment** - Текстовый фрагмент с метаданными. Используется в RAG.

**ToolExecutionRequest** - Представляет запрос на выполнение инструмента.

**UserMessage / AiMessage / SystemMessage** - Типы сообщений в беседе.

## AI/ML Concepts

**Few-Shot Learning** - Предоставление примеров в подсказках. [Модуль 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - ИИ-модели, обученные на огромных объёмах текстовых данных.

**Reasoning Effort** - Параметр GPT-5, управляющий глубиной рассуждений. [Модуль 02](../02-prompt-engineering/README.md)

**Temperature** - Контролирует случайность вывода. Низкая=детерминированно, высокая=креативно.

**Vector Database** - Специализированная база данных для эмбеддингов. [Модуль 03](../03-rag/README.md)

**Zero-Shot Learning** - Выполнение задач без примеров. [Модуль 02](../02-prompt-engineering/README.md)

## Prompt Engineering - [Модуль 02](../02-prompt-engineering/README.md)

**Chain-of-Thought** - Пошаговое рассуждение для повышения точности.

**Constrained Output** - Принуждение к конкретному формату или структуре.

**High Eagerness** - Шаблон GPT-5 для тщательного рассуждения.

**Low Eagerness** - Шаблон GPT-5 для быстрых ответов.

**Multi-Turn Conversation** - Поддержание контекста между обменами.

**Role-Based Prompting** - Установка персонажа модели через системные сообщения.

**Self-Reflection** - Модель оценивает и улучшает свой вывод.

**Structured Analysis** - Фиксированная рамка оценки.

**Task Execution Pattern** - План → Выполнение → Подведение итогов.

## RAG (генерация с дополнением через извлечение) - [Модуль 03](../03-rag/README.md)

**Document Processing Pipeline** - Загрузка → разбиение на фрагменты → встраивание → хранение.

**In-Memory Embedding Store** - Непостоянное хранилище для тестирования.

**RAG** - Сочетает извлечение с генерацией для обоснования ответов.

**Similarity Score** - Мера (0-1) семантического сходства.

**Source Reference** - Метаданные о полученном контенте.

## Agents and Tools - [Модуль 04](../04-tools/README.md)

**@Tool Annotation** - Помечает Java-методы как инструменты, вызываемые ИИ.

**ReAct Pattern** - Рассуждать → Действовать → Наблюдать → Повторять.

**Session Management** - Раздельные контексты для разных пользователей.

**Tool** - Функция, которую AI-агент может вызвать.

**Tool Description** - Документация о назначении инструмента и его параметрах.

## Model Context Protocol (MCP) - [Модуль 05](../05-mcp/README.md)

**MCP** - Стандарт для подключения AI-приложений к внешним инструментам.

**MCP Client** - Приложение, подключающееся к MCP-серверам.

**MCP Server** - Сервис, предоставляющий инструменты через MCP.

**Stdio Transport** - Сервер в виде подпроцесса через stdin/stdout.

**Tool Discovery** - Клиент запрашивает у сервера доступные инструменты.

## Azure Services - [Модуль 01](../01-introduction/README.md)

**Azure AI Search** - Облачный поиск с векторными возможностями. [Модуль 03](../03-rag/README.md)

**Azure Developer CLI (azd)** - Развёртывает ресурсы Azure.

**Azure OpenAI** - Корпоративный AI-сервис Microsoft.

**Bicep** - Язык инфраструктуры как кода для Azure. [Руководство по инфраструктуре](../01-introduction/infra/README.md)

**Deployment Name** - Имя для развертывания модели в Azure.

**GPT-5** - Последняя модель OpenAI с контролем рассуждений. [Модуль 02](../02-prompt-engineering/README.md)
