## Основные понятия

**AI Agent** - Система, использующая ИИ для рассуждений и автономных действий. [Модуль 04](../04-tools/README.md)

**Chain** - Последовательность операций, где вывод подается на следующий шаг.

**Chunking** - Разбиение документов на меньшие части. Обычно: 300-500 токенов с перекрытием. [Модуль 03](../03-rag/README.md)

**Context Window** - Максимальное количество токенов, которые модель может обработать. GPT-5: 400K токенов.

**Embeddings** - Числовые векторы, представляющие смысл текста. [Модуль 03](../03-rag/README.md)

**Function Calling** - Модель генерирует структурированные запросы для вызова внешних функций. [Модуль 04](../04-tools/README.md)

**Hallucination** - Когда модели генерируют неверную, но правдоподобную информацию.

**Prompt** - Текстовый ввод для языковой модели. [Модуль 02](../02-prompt-engineering/README.md)

**Semantic Search** - Поиск по смыслу с использованием эмбеддингов, а не ключевых слов. [Модуль 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: без памяти. Stateful: сохраняет историю разговора. [Модуль 01](../01-introduction/README.md)

**Tokens** - Основные текстовые единицы, которые обрабатывают модели. Влияют на стоимость и лимиты. [Модуль 01](../01-introduction/README.md)

**Tool Chaining** - Последовательное выполнение инструментов, где вывод влияет на следующий вызов. [Модуль 04](../04-tools/README.md)

## Компоненты LangChain4j

**AiServices** - Создает типобезопасные интерфейсы AI-сервисов.

**OpenAiOfficialChatModel** - Унифицированный клиент для моделей OpenAI и Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - Создает эмбеддинги с помощью официального клиента OpenAI (поддерживает OpenAI и Azure OpenAI).

**ChatModel** - Основной интерфейс для языковых моделей.

**ChatMemory** - Поддерживает историю разговора.

**ContentRetriever** - Находит релевантные части документов для RAG.

**DocumentSplitter** - Разбивает документы на части.

**EmbeddingModel** - Преобразует текст в числовые векторы.

**EmbeddingStore** - Хранит и извлекает эмбеддинги.

**MessageWindowChatMemory** - Поддерживает скользящее окно последних сообщений.

**PromptTemplate** - Создает повторно используемые подсказки с заполнителями `{{variable}}`.

**TextSegment** - Текстовый фрагмент с метаданными. Используется в RAG.

**ToolExecutionRequest** - Представляет запрос на выполнение инструмента.

**UserMessage / AiMessage / SystemMessage** - Типы сообщений в разговоре.

## Понятия ИИ/МО

**Few-Shot Learning** - Предоставление примеров в подсказках. [Модуль 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - ИИ-модели, обученные на огромных объемах текста.

**Reasoning Effort** - Параметр GPT-5, контролирующий глубину рассуждений. [Модуль 02](../02-prompt-engineering/README.md)

**Temperature** - Контролирует случайность вывода. Низкая=детерминированный, высокая=креативный.

**Vector Database** - Специализированная база данных для эмбеддингов. [Модуль 03](../03-rag/README.md)

**Zero-Shot Learning** - Выполнение задач без примеров. [Модуль 02](../02-prompt-engineering/README.md)

## Инжиниринг подсказок - [Модуль 02](../02-prompt-engineering/README.md)

**Chain-of-Thought** - Пошаговое рассуждение для повышения точности.

**Constrained Output** - Принудительное соблюдение определенного формата или структуры.

**High Eagerness** - Шаблон GPT-5 для тщательных рассуждений.

**Low Eagerness** - Шаблон GPT-5 для быстрых ответов.

**Multi-Turn Conversation** - Поддержание контекста в нескольких обменах.

**Role-Based Prompting** - Установка персонажа модели через системные сообщения.

**Self-Reflection** - Модель оценивает и улучшает свой вывод.

**Structured Analysis** - Фиксированная структура оценки.

**Task Execution Pattern** - План → Выполнение → Резюме.

## RAG (Генерация с расширением поиска) - [Модуль 03](../03-rag/README.md)

**Document Processing Pipeline** - Загрузка → разбиение → эмбеддинг → хранение.

**In-Memory Embedding Store** - Непостоянное хранилище для тестирования.

**RAG** - Комбинирует поиск и генерацию для обоснованных ответов.

**Similarity Score** - Мера (0-1) семантической схожести.

**Source Reference** - Метаданные о найденном содержимом.

## Агенты и инструменты - [Модуль 04](../04-tools/README.md)

**@Tool Annotation** - Помечает Java-методы как вызываемые ИИ инструменты.

**ReAct Pattern** - Рассуждай → Действуй → Наблюдай → Повторяй.

**Session Management** - Отдельные контексты для разных пользователей.

**Tool** - Функция, которую может вызвать AI агент.

**Tool Description** - Документация о назначении и параметрах инструмента.

## Протокол контекста модели (MCP) - [Модуль 05](../05-mcp/README.md)

**Docker Transport** - MCP сервер в Docker-контейнере.

**MCP** - Стандарт для подключения AI-приложений к внешним инструментам.

**MCP Client** - Приложение, подключающееся к MCP серверам.

**MCP Server** - Сервис, предоставляющий инструменты через MCP.

**Server-Sent Events (SSE)** - Потоковая передача от сервера к клиенту по HTTP.

**Stdio Transport** - Сервер как подпроцесс через stdin/stdout.

**Streamable HTTP Transport** - HTTP с SSE для общения в реальном времени.

**Tool Discovery** - Клиент запрашивает у сервера доступные инструменты.

## Сервисы Azure - [Модуль 01](../01-introduction/README.md)

**Azure AI Search** - Облачный поиск с векторными возможностями. [Модуль 03](../03-rag/README.md)

**Azure Developer CLI (azd)** - Разворачивает ресурсы Azure.

**Azure OpenAI** - Корпоративный AI-сервис Microsoft.

**Bicep** - Язык инфраструктуры как кода для Azure. [Руководство по инфраструктуре](../01-introduction/infra/README.md)

**Deployment Name** - Имя развертывания модели в Azure.

**GPT-5** - Последняя модель OpenAI с контролем рассуждений. [Модуль 02](../02-prompt-engineering/README.md)
