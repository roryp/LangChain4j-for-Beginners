## Основные понятия

**AI Agent** – система, использующая ИИ для рассуждения и автономных действий. [Модуль 04](../04-tools/README.md)

**Chain** – последовательность операций, где выход одного шага является входом следующего.

**Chunking** – разбиение документов на меньшие части. Обычно: 300-500 токенов с перекрытием. [Модуль 03](../03-rag/README.md)

**Context Window** – максимальное количество токенов, обрабатываемых моделью. GPT-5: 400К токенов.

**Embeddings** – числовые векторы, представляющие смысл текста. [Модуль 03](../03-rag/README.md)

**Function Calling** – модель генерирует структурированные запросы для вызова внешних функций. [Модуль 04](../04-tools/README.md)

**Hallucination** – когда модели создают неверную, но правдоподобную информацию.

**Prompt** – текстовый ввод для языковой модели. [Модуль 02](../02-prompt-engineering/README.md)

**Semantic Search** – поиск по смыслу с использованием embedding, а не ключевых слов. [Модуль 03](../03-rag/README.md)

**Stateful vs Stateless** – Stateless: без памяти. Stateful: сохраняет историю диалога. [Модуль 01](../01-introduction/README.md)

**Tokens** – базовые текстовые единицы, которые обрабатывают модели. Влияют на стоимость и лимиты. [Модуль 01](../01-introduction/README.md)

**Tool Chaining** – последовательное выполнение инструментов, где вывод влияет на следующий вызов. [Модуль 04](../04-tools/README.md)

## Компоненты LangChain4j

**AiServices** – создает типобезопасные интерфейсы для сервисов ИИ.

**OpenAiOfficialChatModel** – единый клиент для моделей OpenAI и Azure OpenAI.

**OpenAiOfficialEmbeddingModel** – создает embedding с использованием официального клиента OpenAI (поддерживает и Azure).

**ChatModel** – основной интерфейс языковых моделей.

**ChatMemory** – хранит историю беседы.

**ContentRetriever** – находит релевантные части документа для RAG.

**DocumentSplitter** – разбивает документы на части.

**EmbeddingModel** – преобразует текст в числовые векторы.

**EmbeddingStore** – сохраняет и извлекает embedding.

**MessageWindowChatMemory** – хранит скользящее окно последних сообщений.

**PromptTemplate** – создает повторно используемые подсказки с плейсхолдерами `{{variable}}`.

**TextSegment** – текстовый фрагмент с метаданными. Используется в RAG.

**ToolExecutionRequest** – представляет запрос на выполнение инструмента.

**UserMessage / AiMessage / SystemMessage** – типы сообщений в диалоге.

## Концепции ИИ/МО

**Few-Shot Learning** – предоставление примеров в подсказках. [Модуль 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** – модели ИИ, обученные на больших объемах текстовых данных.

**Reasoning Effort** – параметр GPT-5, контролирующий глубину рассуждений. [Модуль 02](../02-prompt-engineering/README.md)

**Temperature** – контролирует случайность вывода. Низкое — детерминированный, высокое — креативный.

**Vector Database** – специализированная база данных для embedding. [Модуль 03](../03-rag/README.md)

**Zero-Shot Learning** – выполнение задач без примеров. [Модуль 02](../02-prompt-engineering/README.md)

## Защитные механизмы - [Модуль 00](../00-quick-start/README.md)

**Defense in Depth** – многоуровневая защита, сочетающая прикладные guardrails с фильтрами безопасности провайдера.

**Hard Block** – провайдер возвращает ошибку HTTP 400 при серьезных нарушениях контента.

**InputGuardrail** – интерфейс LangChain4j для проверки пользовательского ввода до передачи в LLM. Экономит ресурсы и снижает задержки, блокируя вредные запросы заранее.

**InputGuardrailResult** – возвращаемый тип для валидации guardrail: `success()` или `fatal("причина")`.

**OutputGuardrail** – интерфейс для проверки ответов ИИ перед возвратом пользователю.

**Provider Safety Filters** – встроенные фильтры контента от провайдеров ИИ (например, GitHub Models), ловящие нарушения на уровне API.

**Soft Refusal** – модель вежливо отказывается отвечать без генерации ошибки.

## Агентный модуль - [Модуль 05](../05-mcp/README.md)

**@Agent Annotation** – помечает интерфейсы как ИИ-агентов с декларативным описанием поведения.

**Agent Listener** – хуки для мониторинга выполнения агента через `beforeAgentInvocation()` и `afterAgentInvocation()`.

**Agentic Scope** – общая память, где агенты сохраняют результаты по ключу `outputKey` для последующего использования.

**AgenticServices** – фабрика для создания агентов с помощью `agentBuilder()` и `supervisorBuilder()`.

**Conditional Workflow** – маршрут по условиям к разным специализированным агентам.

**Human-in-the-Loop** – паттерн с участием человека для одобрения или проверки контента.

**langchain4j-agentic** – зависимость Maven для декларативного создания агентов (экспериментально).

**Loop Workflow** – повторение вызова агента до достижения условия (например, оценка качества ≥ 0.8).

**outputKey** – параметр аннотации агента, указывающий где в Agentic Scope сохранять результаты.

**Parallel Workflow** – параллельное выполнение нескольких агентов для независимых задач.

**Response Strategy** – способ формулировки итогового ответа супервизором: LAST, SUMMARY, или SCORED.

**Sequential Workflow** – последовательное выполнение агентов, где вывод одного шага идет на следующий.

**Supervisor Agent Pattern** – продвинутый паттерн, в котором супервизор LLM динамически решает, каких подагентов вызвать.

## Протокол контекста модели (MCP) - [Модуль 05](../05-mcp/README.md)

**langchain4j-mcp** – зависимость Maven для интеграции MCP в LangChain4j.

**MCP** – Model Context Protocol: стандарт для подключения ИИ-приложений к внешним инструментам. Напиши один раз – используй везде.

**MCP Client** – приложение, которое подключается к MCP-серверам для обнаружения и использования инструментов.

**MCP Server** – сервис, предоставляющий инструменты через MCP с описаниями и схемами параметров.

**McpToolProvider** – компонент LangChain4j, оборачивающий MCP-инструменты для использования в сервисах ИИ и агентах.

**McpTransport** – интерфейс для общения MCP. Реализации включают Stdio и HTTP.

**Stdio Transport** – локальный транспорт процессов через stdin/stdout. Полезно для доступа к файловой системе или командным утилитам.

**StdioMcpTransport** – реализация LangChain4j, запускающая MCP-сервер как подпроцесс.

**Tool Discovery** – клиент запрашивает у сервера список доступных инструментов с описаниями и схемами.
