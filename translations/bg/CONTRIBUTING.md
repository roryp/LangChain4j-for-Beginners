# Contributing

Благодарим ви за интереса да допринесете! Ние приветстваме pull заявки и подобрения на този курс. Моля, следвайте тези насоки при подаване на промени:

1. **Без тайни в комитите.** Никога не комитвайте API ключове или какъвто и да е конфиденциален материал. Използвайте локален `.env` за собствено тестване.
2. **Дръжте промените фокусирани.** Малки, ограничени pull заявки са по-лесни за преглед и поддръжка. Ако поправяте няколко проблема, моля подавайте ги отделно.
3. **Добавяйте тестове, когато е възможно.** Unit или интеграционни тестове помагат да се хванат регресии. За Spring Boot модулите включвайте тестове в папката `src/test/java`.
4. **Следвайте конвенциите за стил на кода.** Използвайте съществуващата структура на проекта като ръководство. Разчитаме на Maven за управление на зависимости и на Spring Boot Maven плъгина за пакетиране на приложения.

Благодарим ви, че помагате този курс да стане по-добър!
