# Bidra

Takk for din interesse for å bidra! Vi ønsker pull requests og forbedringer til dette kurset velkommen. Vennligst følg disse retningslinjene når du sender inn endringer:

1. **Ingen hemmeligheter i commits.** Ikke legg inn API-nøkler eller noe konfidensielt materiale. Bruk en lokal `.env` for din egen testing.
2. **Hold endringer fokuserte.** Små, avgrensede pull requests er enklere å gjennomgå og vedlikeholde. Hvis du fikser flere problemer, vennligst send dem inn separat.
3. **Legg til tester der det er mulig.** Enhets- eller integrasjonstester hjelper med å fange opp regresjoner. For Spring Boot-moduler, inkluder tester under `src/test/java`-mappen.
4. **Følg kode-stilkonvensjoner.** Bruk den eksisterende prosjektstrukturen som en veiledning. Vi stoler på Maven for å håndtere avhengigheter og Spring Boot Maven-plugin for å pakke applikasjoner.

Takk for at du hjelper til med å gjøre dette kurset bedre!
