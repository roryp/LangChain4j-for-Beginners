# Contributing

కృతజ్ఞతలు మీ ఆసక్తికి సహకరించడానికి! మేము ఈ కోర్సుకు పుల్ రిక్వెస్టులు మరియు మెరుగుదలలను స్వాగతిస్తాము. మార్పులను సమర్పించేటప్పుడు ఈ మార్గదర్శకాలను అనుసరించండి:

1. **కమిట్లలో రహస్యాలు ఉండకూడదు.** ఎప్పుడూ API కీలు లేదా ఏదైనా గోప్యమైన పదార్థాన్ని కమిట్ చేయవద్దు. మీ స్వంత పరీక్ష కోసం స్థానిక `.env` ఉపయోగించండి.
2. **మార్పులను కేంద్రీకృతంగా ఉంచండి.** చిన్న, పరిమిత పుల్ రిక్వెస్టులు సమీక్షించడానికి మరియు నిర్వహించడానికి సులభం. మీరు అనేక సమస్యలను పరిష్కరిస్తున్నట్లయితే, దయచేసి వాటిని వేర్వేరు సమర్పించండి.
3. **సాధ్యమైన చోట పరీక్షలను జోడించండి.** యూనిట్ లేదా ఇంటిగ్రేషన్ పరీక్షలు పునరావృతాలను పట్టుకోవడంలో సహాయపడతాయి. స్ప్రింగ్ బూట్ మాడ్యూల్స్ కోసం, `src/test/java` ఫోల్డర్ కింద పరీక్షలను చేర్చండి.
4. **కోడ్ శైలి నియమాలను అనుసరించండి.** ప్రస్తుత ప్రాజెక్ట్ నిర్మాణాన్ని మార్గదర్శకంగా ఉపయోగించండి. మేము Maven ను ఆధారాలు నిర్వహించడానికి మరియు స్ప్రింగ్ బూట్ Maven ప్లగిన్ ను అప్లికేషన్లను ప్యాకేజింగ్ చేయడానికి ఆధారంగా తీసుకుంటాము.

ఈ కోర్సును మెరుగుపరచడంలో సహాయం చేసినందుకు ధన్యవాదాలు!
