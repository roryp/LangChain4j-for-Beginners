# Contributing

অংশগ্রহণের জন্য আপনার আগ্রহের জন্য ধন্যবাদ! আমরা এই কোর্সে পুল রিকোয়েস্ট এবং উন্নতি স্বাগত জানাই। পরিবর্তন জমা দেওয়ার সময় দয়া করে এই নির্দেশিকাগুলি অনুসরণ করুন:

1. **কমিটে কোনো গোপনীয়তা থাকবে না।** কখনোই API কী বা কোনো গোপনীয় উপাদান কমিট করবেন না। আপনার নিজের পরীক্ষার জন্য একটি স্থানীয় `.env` ব্যবহার করুন।
2. **পরিবর্তনগুলি কেন্দ্রীভূত রাখুন।** ছোট, সীমাবদ্ধ পুল রিকোয়েস্টগুলি পর্যালোচনা এবং রক্ষণাবেক্ষণের জন্য সহজ। আপনি যদি একাধিক সমস্যা ঠিক করছেন, তবে দয়া করে সেগুলি আলাদাভাবে জমা দিন।
3. **যেখানে সম্ভব সেখানে টেস্ট যোগ করুন।** ইউনিট বা ইন্টিগ্রেশন টেস্টগুলি রিগ্রেশন ধরতে সাহায্য করে। Spring Boot মডিউলগুলির জন্য, `src/test/java` ফোল্ডারের অধীনে টেস্ট অন্তর্ভুক্ত করুন।
4. **কোড স্টাইল কনভেনশন অনুসরণ করুন।** বিদ্যমান প্রকল্প কাঠামোকে নির্দেশিকা হিসেবে ব্যবহার করুন। আমরা Maven-কে নির্ভরশীলতা পরিচালনার জন্য এবং Spring Boot Maven প্লাগইনকে অ্যাপ্লিকেশন প্যাকেজ করার জন্য নির্ভর করি।

এই কোর্সটিকে আরও ভালো করতে সাহায্য করার জন্য ধন্যবাদ!
