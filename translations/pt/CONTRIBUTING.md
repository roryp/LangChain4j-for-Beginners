# Contribuir

Obrigado pelo seu interesse em contribuir! Agradecemos pedidos de pull
e melhorias para este curso. Por favor, siga estas
diretrizes ao submeter alterações:

1. **Sem segredos nos commits.** Nunca faça commit de chaves API ou qualquer
   material confidencial. Use um `.env` local para os seus próprios testes.
2. **Mantenha as alterações focadas.** Pedidos de pull pequenos e específicos são mais fáceis
   de rever e manter. Se estiver a corrigir múltiplos problemas, por favor
   submeta-os separadamente.
3. **Adicione testes sempre que possível.** Testes unitários ou de integração ajudam
   a evitar regressões. Para módulos Spring Boot, inclua testes na pasta
   `src/test/java`.
4. **Siga as convenções de estilo de código.** Use a estrutura existente do projeto
   como guia. Confiamos no Maven para gerir dependências e no plugin Spring Boot Maven para empacotar aplicações.

Obrigado por ajudar a melhorar este curso!
