# Contributing

योगदान दिनु भएकोमा धन्यवाद! हामी यस कोर्समा पुल अनुरोधहरू र सुधारहरू स्वागत गर्दछौं। कृपया परिवर्तनहरू पेश गर्दा यी दिशानिर्देशहरू पालना गर्नुहोस्:

1. **कमिटहरूमा गोप्य कुरा नराख्नुहोस्।** कहिल्यै API कुञ्जीहरू वा कुनै गोप्य सामग्री कमिट नगर्नुहोस्। आफ्नो परीक्षणका लागि स्थानीय `.env` प्रयोग गर्नुहोस्।
2. **परिवर्तनहरू केन्द्रित राख्नुहोस्।** साना, सीमित पुल अनुरोधहरू समीक्षा र मर्मत गर्न सजिलो हुन्छ। यदि तपाईं धेरै समस्याहरू समाधान गर्दै हुनुहुन्छ भने, कृपया तिनीहरूलाई अलग-अलग पेश गर्नुहोस्।
3. **जहाँ सम्भव छ परीक्षणहरू थप्नुहोस्।** युनिट वा एकीकरण परीक्षणहरूले पुनरावृत्तिहरू पत्ता लगाउन मद्दत गर्छ। स्प्रिंग बुट मोड्युलहरूको लागि, `src/test/java` फोल्डर भित्र परीक्षणहरू समावेश गर्नुहोस्।
4. **कोड शैलीका नियमहरू पालना गर्नुहोस्।** विद्यमान परियोजना संरचनालाई मार्गदर्शनको रूपमा प्रयोग गर्नुहोस्। हामी निर्भरता व्यवस्थापनका लागि Maven र एप्लिकेसन प्याकेज गर्न स्प्रिंग बुट Maven प्लगइनमा निर्भर गर्दछौं।

यस कोर्सलाई अझ राम्रो बनाउन सहयोग पुर्‍याउनु भएकोमा धन्यवाद!
