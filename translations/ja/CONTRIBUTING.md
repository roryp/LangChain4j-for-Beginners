# Contributing

ご協力いただきありがとうございます！このコースへのプルリクエストや改善提案を歓迎します。変更を提出する際は、以下のガイドラインに従ってください。

1. **コミットに秘密情報を含めないこと。** APIキーや機密情報をコミットしないでください。ご自身のテストにはローカルの `.env` を使用してください。
2. **変更は焦点を絞ること。** 小さく範囲を限定したプルリクエストの方がレビューやメンテナンスが容易です。複数の問題を修正する場合は、別々に提出してください。
3. **可能な限りテストを追加すること。** ユニットテストや統合テストは回帰を防ぐのに役立ちます。Spring Bootモジュールの場合は、`src/test/java` フォルダー内にテストを含めてください。
4. **コードスタイルの規約に従うこと。** 既存のプロジェクト構成をガイドとして使用してください。依存関係の管理にはMavenを、アプリケーションのパッケージングにはSpring Boot Mavenプラグインを使用しています。

このコースをより良くするためにご協力いただき、ありがとうございます！
