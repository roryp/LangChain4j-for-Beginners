# Contributing

Vielen Dank für Ihr Interesse an einer Mitarbeit! Wir freuen uns über Pull Requests und Verbesserungen für diesen Kurs. Bitte beachten Sie diese Richtlinien beim Einreichen von Änderungen:

1. **Keine Geheimnisse in Commits.** Niemals API-Schlüssel oder vertrauliche Materialien committen. Verwenden Sie eine lokale `.env` für Ihre eigenen Tests.
2. **Änderungen fokussiert halten.** Kleine, gezielte Pull Requests sind leichter zu überprüfen und zu pflegen. Wenn Sie mehrere Probleme beheben, reichen Sie diese bitte separat ein.
3. **Fügen Sie nach Möglichkeit Tests hinzu.** Unit- oder Integrationstests helfen, Regressionen zu vermeiden. Für Spring Boot-Module fügen Sie Tests im Ordner `src/test/java` hinzu.
4. **Befolgen Sie die Code-Stilkonventionen.** Verwenden Sie die bestehende Projektstruktur als Leitfaden. Wir verlassen uns auf Maven zur Verwaltung der Abhängigkeiten und das Spring Boot Maven Plugin zum Packen der Anwendungen.

Vielen Dank, dass Sie dazu beitragen, diesen Kurs zu verbessern!
