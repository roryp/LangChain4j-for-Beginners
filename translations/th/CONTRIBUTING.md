# Contributing

ขอขอบคุณที่สนใจมีส่วนร่วม! เราต้อนรับคำขอ pull และการปรับปรุงหลักสูตรนี้ โปรดปฏิบัติตามแนวทางเหล่านี้เมื่อส่งการเปลี่ยนแปลง:

1. **ไม่เก็บความลับใน commits.** ห้าม commit คีย์ API หรือข้อมูลลับใดๆ ใช้ `.env` ในเครื่องของคุณสำหรับการทดสอบส่วนตัว
2. **รักษาการเปลี่ยนแปลงให้เน้นจุดเดียว.** คำขอ pull ที่เล็กและมีขอบเขตชัดเจนจะง่ายต่อการตรวจสอบและดูแล หากคุณแก้ไขหลายปัญหา โปรดส่งแยกกัน
3. **เพิ่มการทดสอบเมื่อเป็นไปได้.** การทดสอบหน่วยหรือการทดสอบแบบบูรณาการช่วยจับข้อผิดพลาด สำหรับโมดูล Spring Boot ให้รวมการทดสอบไว้ในโฟลเดอร์ `src/test/java`
4. **ปฏิบัติตามรูปแบบโค้ด.** ใช้โครงสร้างโปรเจกต์ที่มีอยู่เป็นแนวทาง เราใช้ Maven ในการจัดการ dependencies และปลั๊กอิน Spring Boot Maven ในการแพ็กเกจแอปพลิเคชัน

ขอขอบคุณที่ช่วยทำให้หลักสูตรนี้ดีขึ้น!
