# Contributing

תודה על ההתעניינות שלך בתרומה! אנו מקבלים בברכה בקשות משיכה ושיפורים לקורס זה. אנא עקוב אחר ההנחיות הבאות בעת הגשת שינויים:

1. **אין סודות בקומיטים.** לעולם אל תבצע קומיט של מפתחות API או כל חומר סודי אחר. השתמש בקובץ `.env` מקומי לצורך הבדיקות שלך.
2. **שמור על שינויים ממוקדים.** בקשות משיכה קטנות וממוקדות קלות יותר לסקירה ולתחזוקה. אם אתה מתקן מספר בעיות, אנא הגש אותן בנפרד.
3. **הוסף בדיקות כאשר אפשרי.** בדיקות יחידה או אינטגרציה עוזרות ללכוד נסיגות. עבור מודולים של Spring Boot, כלול בדיקות תחת התיקייה `src/test/java`.
4. **עקוב אחר קונבנציות סגנון הקוד.** השתמש במבנה הפרויקט הקיים כמדריך. אנו מסתמכים על Maven לניהול תלותים ועל תוסף Spring Boot Maven לאריזת יישומים.

תודה שעזרת להפוך את הקורס הזה לטוב יותר!
