# Contributing

Cảm ơn bạn đã quan tâm đóng góp! Chúng tôi hoan nghênh các pull request và cải tiến cho khóa học này. Vui lòng tuân theo các hướng dẫn sau khi gửi thay đổi:

1. **Không để lộ thông tin bí mật trong các commit.** Không bao giờ commit khóa API hoặc bất kỳ tài liệu bảo mật nào. Sử dụng `.env` cục bộ cho việc thử nghiệm của bạn.
2. **Giữ các thay đổi tập trung.** Các pull request nhỏ, có phạm vi rõ ràng sẽ dễ dàng hơn để xem xét và duy trì. Nếu bạn sửa nhiều vấn đề, vui lòng gửi chúng riêng biệt.
3. **Thêm các bài kiểm tra khi có thể.** Các bài kiểm tra đơn vị hoặc tích hợp giúp phát hiện lỗi hồi quy. Đối với các module Spring Boot, bao gồm các bài kiểm tra trong thư mục `src/test/java`.
4. **Tuân theo quy ước phong cách mã.** Sử dụng cấu trúc dự án hiện có làm hướng dẫn. Chúng tôi dựa vào Maven để quản lý các phụ thuộc và plugin Spring Boot Maven để đóng gói ứng dụng.

Cảm ơn bạn đã giúp làm cho khóa học này tốt hơn!
