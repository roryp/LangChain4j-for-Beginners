# Panustamine

Täname, et olete huvitatud panustamisest! Me tervitame pull-päringuid ja selle kursuse täiustusi. Palun järgige muudatuste esitamisel järgmisi juhiseid:

1. **Ärge pange salajasi andmeid commit'idesse.** Ärge kunagi pange API võtmeid ega muid konfidentsiaalseid materjale commit'i. Kasutage oma testimiseks kohalikku `.env` faili.
2. **Hoidke muudatused fookuses.** Väikesed, piiratud pull-päringud on lihtsamad üle vaadata ja hooldada. Kui parandate mitut probleemi, esitage need palun eraldi.
3. **Lisage testid, kui võimalik.** Ühik- või integratsioonitestid aitavad regressioone tabada. Spring Boot moodulite puhul lisage testid kausta `src/test/java`.
4. **Järgige koodi stiili konventsioone.** Kasutage olemasolevat projekti struktuuri juhisena. Me kasutame sõltuvuste haldamiseks Mavenit ja rakenduste pakendamiseks Spring Boot Maven pluginat.

Täname, et aitate selle kursuse paremaks muuta!
