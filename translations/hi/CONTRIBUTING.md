# Contributing

योगदान देने में आपकी रुचि के लिए धन्यवाद! हम इस कोर्स में पुल रिक्वेस्ट और सुधारों का स्वागत करते हैं। कृपया परिवर्तन सबमिट करते समय इन दिशानिर्देशों का पालन करें:

1. **कमिट्स में कोई रहस्य न रखें।** कभी भी API कुंजी या कोई गोपनीय सामग्री कमिट न करें। अपने स्वयं के परीक्षण के लिए स्थानीय `.env` का उपयोग करें।
2. **परिवर्तनों को केंद्रित रखें।** छोटे, सीमित पुल रिक्वेस्ट की समीक्षा और रखरखाव करना आसान होता है। यदि आप कई मुद्दों को ठीक कर रहे हैं, तो कृपया उन्हें अलग-अलग सबमिट करें।
3. **जहां संभव हो, परीक्षण जोड़ें।** यूनिट या इंटीग्रेशन परीक्षण रिग्रेशन पकड़ने में मदद करते हैं। Spring Boot मॉड्यूल के लिए, `src/test/java` फ़ोल्डर के अंतर्गत परीक्षण शामिल करें।
4. **कोड शैली कन्वेंशनों का पालन करें।** मौजूदा प्रोजेक्ट संरचना को मार्गदर्शक के रूप में उपयोग करें। हम निर्भरताओं को प्रबंधित करने के लिए Maven और एप्लिकेशन पैकेज करने के लिए Spring Boot Maven प्लगइन पर निर्भर करते हैं।

इस कोर्स को बेहतर बनाने में मदद करने के लिए धन्यवाद!
