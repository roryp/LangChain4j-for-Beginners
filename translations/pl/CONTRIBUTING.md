# Contributing

Dziękujemy za zainteresowanie wniesieniem wkładu! Zachęcamy do przesyłania pull requestów i ulepszeń do tego kursu. Prosimy o przestrzeganie poniższych wytycznych podczas zgłaszania zmian:

1. **Brak sekretów w commitach.** Nigdy nie commituj kluczy API ani żadnych poufnych materiałów. Do własnych testów używaj lokalnego pliku `.env`.
2. **Utrzymuj zmiany skupione.** Małe, wyodrębnione pull requesty są łatwiejsze do przeglądu i utrzymania. Jeśli naprawiasz wiele problemów, prosimy zgłaszać je osobno.
3. **Dodawaj testy, jeśli to możliwe.** Testy jednostkowe lub integracyjne pomagają wykrywać regresje. Dla modułów Spring Boot dołącz testy w folderze `src/test/java`.
4. **Przestrzegaj konwencji stylu kodu.** Korzystaj z istniejącej struktury projektu jako wskazówki. Polegamy na Maven do zarządzania zależnościami oraz na wtyczce Spring Boot Maven do pakowania aplikacji.

Dziękujemy za pomoc w ulepszaniu tego kursu!
