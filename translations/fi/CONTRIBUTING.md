# Contributing

Kiitos mielenkiinnostasi osallistua! Otamme mielellämme vastaan pull
pyyntöjä ja parannuksia tähän kurssiin. Noudata näitä
ohjeita tehdessäsi muutoksia:

1. **Ei salaisuuksia commiteissa.** Älä koskaan commitoi API-avaimia tai mitään
   luottamuksellista materiaalia. Käytä paikallista `.env`-tiedostoa omaan testaukseesi.
2. **Pidä muutokset kohdennettuina.** Pienet, rajatut pull pyynnöt ovat helpompia
   tarkistaa ja ylläpitää. Jos korjaat useita ongelmia, lähetä ne erikseen.
3. **Lisää testejä mahdollisuuksien mukaan.** Yksikkö- tai integraatiotestit auttavat
   havaitsemaan takapakkeja. Spring Boot -moduuleissa lisää testit `src/test/java`-kansioon.
4. **Noudata koodityylikonventioita.** Käytä olemassa olevaa projektirakennetta ohjeena.
   Luotamme Mavenin riippuvuuksien hallintaan ja Spring Boot Maven -pluginin sovellusten pakkaamiseen.

Kiitos, että autat tekemään tästä kurssista paremman!
