# ಕೊಡುಗೆ ನೀಡುವುದು

ಕೊಡುಗೆ ನೀಡಲು ನಿಮ್ಮ ಆಸಕ್ತಿಗೆ ಧನ್ಯವಾದಗಳು! ನಾವು ಈ ಕೋರ್ಸ್‌ಗೆ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ಗಳು ಮತ್ತು ಸುಧಾರಣೆಗಳನ್ನು ಸ್ವಾಗತಿಸುತ್ತೇವೆ. ದಯವಿಟ್ಟು ಬದಲಾವಣೆಗಳನ್ನು ಸಲ್ಲಿಸುವಾಗ ಈ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಿ:

1. **ಕಮಿಟ್‌ಗಳಲ್ಲಿ ರಹಸ್ಯಗಳನ್ನು ಇರಿಸಬೇಡಿ.** API ಕೀಗಳು ಅಥವಾ ಯಾವುದೇ ಗುಪ್ತ ಮಾಹಿತಿಯನ್ನು ಎಂದಿಗೂ ಕಮಿಟ್ ಮಾಡಬೇಡಿ. ನಿಮ್ಮ ಸ್ವಂತ ಪರೀಕ್ಷೆಗೆ ಸ್ಥಳೀಯ `.env` ಅನ್ನು ಬಳಸಿ.
2. **ಬದಲಾವಣೆಗಳನ್ನು ಕೇಂದ್ರೀಕರಿಸಿ.** ಸಣ್ಣ, ಗುರಿತಗೊಂಡ ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸುಲಭ. ನೀವು ಹಲವಾರು ಸಮಸ್ಯೆಗಳನ್ನು ಸರಿಪಡಿಸುತ್ತಿದ್ದರೆ, ದಯವಿಟ್ಟು ಅವುಗಳನ್ನು ಪ್ರತ್ಯೇಕವಾಗಿ ಸಲ್ಲಿಸಿ.
3. **ಸಾಧ್ಯವಾದಲ್ಲಿ ಪರೀಕ್ಷೆಗಳನ್ನು ಸೇರಿಸಿ.** ಯೂನಿಟ್ ಅಥವಾ ಇಂಟಿಗ್ರೇಶನ್ ಪರೀಕ್ಷೆಗಳು ಹಿಂಪಡೆಯುವಿಕೆಗಳನ್ನು ಹಿಡಿಯಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ. ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಮೋಡ್ಯೂಲ್‌ಗಳಿಗಾಗಿ, `src/test/java` ಫೋಲ್ಡರ್ ಅಡಿಯಲ್ಲಿ ಪರೀಕ್ಷೆಗಳನ್ನು ಸೇರಿಸಿ.
4. **ಕೋಡ್ ಶೈಲಿ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸಿ.** ಇತ್ತೀಚಿನ ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆಯನ್ನು ಮಾರ್ಗದರ್ಶಕವಾಗಿ ಬಳಸಿ. ನಾವು ಅವಲಂಬನೆಗಳನ್ನು ನಿರ್ವಹಿಸಲು Maven ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪ್ಯಾಕೇಜ್ ಮಾಡಲು ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ Maven ಪ್ಲಗಿನ್ ಅನ್ನು ಅವಲಂಬಿಸುತ್ತೇವೆ.

ಈ ಕೋರ್ಸ್ ಅನ್ನು ಉತ್ತಮಗೊಳಿಸಲು ಸಹಾಯ ಮಾಡಿದಕ್ಕಾಗಿ ಧನ್ಯವಾದಗಳು!
