## ಮೂಲಿಕ ತತ್ವಗಳು

**ಎಐ ಏಜೆಂಟ್** - ಸ್ವತಂತ್ರವಾಗಿ ಯುಕ್ತಿವಾದ ಮತ್ತು ಕ್ರಿಯಾತ್ಮಕವಾಗಿರುವ ಎಐ ಉಪಯೋಗಿಸುವ ವ್ಯವಸ್ಥೆ. [ಮොಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಚೈನ್** - ಔಟ್‌ಪುಟ್ ಮುಂದಿನ ಹಂತಕ್ಕೆ ಕೊಡಬರುತ್ತಿರುವ ಕಾರ್ಯ ಕ್ರಮ.

**ಚಂಕಿಂಗ್** - ದಸ್ತಾವೇಜುಗಳನ್ನು ಚಿಕ್ಕ ಚುಕ್ಕಾಣುಗಳಲ್ಲಿ ಕಲ್ಲಿಸುವುದು. ಸಾಮಾನ್ಯ: 300-500 ಟೋಕನ್ ಗಳು, ಸಮಾನಾಂತರದೊಂದಿಗೆ. [ಮೊಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋ** - ಮಾಡೆಲ್ ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದಾದ ಗರಿಷ್ಠ ಟೋಕನ್ಗಳು. GPT-5: 400K ಟೋಕನ್ಗಳು.

**ಎಂಬೆಡ್ಡಿಂಗ್‌ಗಳು** - ಪಠ್ಯದ ಅರ್ಥವನ್ನು ಪ್ರತಿಬಿಂಬಿಸುವ ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್‌ಗಳು. [ಮೊಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್** - ಮಾಡೆಲ್ ಹೊರಗಿನ ಕಾರ್ಯಗಳನ್ನು ಕರೆಯಲು ರಚನೆಬದ್ಧ ವಿನಂತಿಗಳನ್ನು ಸೃಷ್ಟಿಸುವುದು. [ಮೊಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಹಾಲುಸಿನೇಷನ್** - ಮಾಡೆಲ್‌ಗಳು ತಪ್ಪು ಆದರೆ ಸಹಜವೂ ಆಗಿರುವ ಮಾಹಿತಿಯನ್ನು ಸೃಷ್ಟಿಸುವಾಗ.

**ಪ್ರಾಂಪ್ಟ್** - ಭಾಷಾ ಮಾಡೆಲ್‌ಗೆ ಇನ್‌ಪುಟ್ ಪಠ್ಯ. [ಮೊಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಸೇಮ್ಯಾಂಟಿಕ್ ಶೋಧ** - ಕೀವರ್ಡ್‌ಗಳ ಬದಲಾಗಿ ಅರ್ಥ ಆಧಾರಿತ ಹುಡುಕು. [ಮೊಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಸ್ಟೇಟ್‌ಫುಲ್ vs ಸ್ಟೇಟ್‌ಲೆಸ್** - ಸ್ಟೇಟ್‌ಲೆಸ್: ಮೆಮರಿ ಇಲ್ಲ. ಸ್ಟೇಟ್‌ಫುಲ್: ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಉಳಿಸುವುದು. [ಮೊಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೋಕನ್ಗಳು** - ಮಾಡೆಲ್‌ಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಪಠ್ಯ ಘಟಕಗಳು. ವೆಚ್ಚ ಮತ್ತು ಮಿತಿಗಳನ್ನು ಪ್ರಭಾವಿಸುತ್ತದೆ. [ಮೊಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೂಲ್ ಚೈನಿಂಗ್** - ಔಟ್‌ಪುಟ್ ಮುಂದಿನ ಕರೆಗಾಗಿ ಉಪಕರಣ ಸರಣಿಗೆ ನಮೂದಿಸುವುದು. [ಮೊಡ್ಯೂಲ್ 04](../04-tools/README.md)

## ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಘಟಕಗಳು

**AiServices** - ಪ್ರಕಾರ-ಸುರಕ್ಷಿತ ಎಐ ಸೇವಾ ಇಂಟರ್ಫೇಸ್ಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

**OpenAiOfficialChatModel** - OpenAI ಮತ್ತು ಅದ್ಬವದ ಎಐ ಮಾಡೆಲ್‌ಗಳಿಗಾಗಿ ಏಕೀಕೃತ ಕ್ಲೈಂಟ್.

**OpenAiOfficialEmbeddingModel** - OpenAI ಅಧಿಕೃತ ಕ್ಲೈಂಟ್ ಉಪಯೋಗಿಸಿ ಎಂಬೆಡ್ಡಿಂಗ್‌ಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ (OpenAI ಮತ್ತು Azure OpenAI ಎರಡಕ್ಕೂ ಬೆಂಬಲ).

**ChatModel** - ಭಾಷಾ ಮಾಡೆಲ್ ಗಳಿಗಾಗಿ ಮೂಲ ಇಂಟರ್ಫೇಸ್.

**ChatMemory** - ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಕಾಯುವುದಕ್ಕೆ.

**ContentRetriever** - RAG ಗೆ ಸಂಬಂಧಿಸಿದ ದಾಖಲೆ ಚುಂಕ್ ಗಳನ್ನು ಹುಡುಕುತ್ತದೆ.

**DocumentSplitter** - ದಸ್ತಾವೇಜುಗಳನ್ನು ಚುಂಕ್ ಗಳಲ್ಲಿ ವಿಭಜಿಸುತ್ತದೆ.

**EmbeddingModel** - ಪಠ್ಯವನ್ನು ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್ ಗಳಾಗಿ ಪರಿವರ್ತಿಸುತ್ತದೆ.

**EmbeddingStore** - ಎಂಬೆಡ್ಡಿಂಗ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸುತ್ತದೆ ಮತ್ತು ಪಡೆದುಕೊಂಡುಹೋಗುತ್ತದೆ.

**MessageWindowChatMemory** - ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ಕಾಯುತ್ತದೆ.

**PromptTemplate** - `{{variable}}` ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್ ಗಳೊಂದಿಗೆ ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್ ಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ.

**TextSegment** - ಮೆಟಾಡೇಟಾ ಜೊತೆಗೆ ಪಠ್ಯದ ಚುಂಕ್. RAG ನಲ್ಲಿ ಬಳಕೆ.

**ToolExecutionRequest** - ಉಪಕರಣ ನಿರ್ವಹಣೆ ವಿನಂತಿಯೊಬ್ಬ ತೋರುತ್ತದೆ.

**UserMessage / AiMessage / SystemMessage** - ಸಂವಾದ ಸಂದೇಶ ಪ್ರಕಾರಗಳು.

## ಎಐ/ಎಂಎಲ್ ತತ್ವಗಳು

**ಫ್ಯೂ-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಪ್ರಾಂಪ್ಟ್ ಗಳಲ್ಲಿ ಉದಾಹರಣೆಗಳನ್ನು ಒದಗಿಸುವುದು. [ಮೊಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾಡೆಲ್ (LLM)** - ಅಪಾರ ಪಠ್ಯ ಡೇಟಾದ ಮೇಲೆ ತರಬೇತಿ ಪಡೆದ ಎಐ ಮಾಡೆಲ್‌ಗಳು.

**ಯುಕ್ತಿವಾದ ಪ್ರಯತ್ನ** - ಚಿಂತನೆ ಆಳವನ್ನು ನಿಯಂತ್ರಿಸುವ GPT-5 ಪರಿಮಿತಿ. [ಮೊಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ತಾಪಮಾನ** - ಔಟ್‌ಪುಟ್ ಯಾದೃಚ್ಛತೆಯನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ. ಕಡಿಮೆ=ನಿಶ್ಚಿತ, ಹೆಚ್ಚು=ಸೃಜನಶೀಲ.

**ವೆಕ್ಟರ್ ಡೇಟಾಬೇಸ್** - ಎಂಬೆಡ್ಡಿಂಗ್‌ಗಳಿಗೆ ವಿಶೇಷ ಡೇಟಾಬೇಸ್. [ಮೊಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಜೀರೊ-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ಕಾರ್ಯ ನಿರ್ವಹಣೆ. [ಮೊಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

## ರಕ್ಷಾಕವಚಗಳು - [ಮೊಡ್ಯೂಲ್ 00](../00-quick-start/README.md)

**ಡಿಫೆನ್ಸ್ ಇನ್ ಡೆಪ್ತ್** - ಅನೇಕ ಹಂತಗಳ ಭದ್ರತಾ ವಿಧಾನ, ಅಪ್ಲಿಕೇಶನ್ ಲೆವೆಲ್ ರಕ್ಷಾಕವಚಗಳನ್ನು ಮತ್ತು ಪೂರೈಕೆದಾರರ ಸುರಕ್ಷತಾ ಫಿಲ್ಟರ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸುವುದು.

**ಹಾರ್ಡ್ ಬ್ಲಾಕ್** - ಗಂಭೀರ ವಿಷಯ ಉಲ್ಲಂಘನೆಗೆ ಪೂರೈಕೆದಾರ HTTP 400 ದೋಷ ಏರ್ಪಡಿಸುವುದು.

**InputGuardrail** - LLMಗೆ ಮುನ್ನ ಬಳಕೆದಾರ ಇನ್‌ಪುಟ್ ಪರಿಶೀಲಿಸುವ ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಇಂಟರ್ಫೇಸ್. ಹಾನಿಕಾರಕ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಪ್ರಾರಂಭದಲ್ಲಿ ತಡೆಹಿಡಿದು ವೆಚ್ಚ ಮತ್ತು ವಿಳಂಬ ಸೇರುವಿಕೆ ಉಳಿಸುತ್ತದೆ.

**InputGuardrailResult** - ಗಾರ್ಡ್ರೈಲ್ ಪರಿಶೀಲನೆಗೆ ಮರಳುವ ಪ್ರಕಾರ: `success()` ಅಥವಾ `fatal("ಕಾರಣ")`.

**OutputGuardrail** - ಬಳಕೆದಾರರಿಗೆ ಮರಳಿ ನೀಡುವ ಮೊದಲು ಎಐ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಪರಿಶೀಲನೆಗೆ ಇಂಟರ್ಫೇಸ್.

**Provider Safety Filters** - ಎಐ ಪೂರೈಕೆದಾರರಿಂದ (ಉದಾ. GitHub Models) ನಿರ್ಮಿತ ಅಂತರಂಗ ಫಿಲ್ಟರ್‌ಗಳು, API ಮಟ್ಟದಲ್ಲಿ ಉಲ್ಲಂಘನೆಗಳನ್ನು ಹಿಡಿಯುತ್ತವೆ.

**Soft Refusal** - ಮಾಡೆಲ್ ದಯವಿಟ್ಟು ಉತ್ತರ ನೀಡಲು ನಿರಾಕರಿಸುವುದು, ದೋಷ ತೋರಿಸುವುದಿಲ್ಲ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ - [ಮೊಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಚೈನ್-ಆಫ್-ಥಾಟ್** - ಉತ್ತಮ ನಿಖರತೆಗೆ ಹಂತ ಹಂತವಾದ ಯುಕ್ತಿವಾದ.

**ನಿಯಂತ್ರಿತ ಔಟ್‌ಪುಟ್** - ನಿರ್ದಿಷ್ಟ ಫಾರ್ಮ್ಯಾಟ್ ಅಥವಾ ರಚನೆ ಅನುಸರಿಸುವುದು.

**ಹೆಚ್ಚು ಉತ್ಸಾಹ** - ಸಮಗ್ರ ಯುಕ್ತಿವಾದಕ್ಕಾಗಿ GPT-5 ಮಾದರಿ.

**ಕಡಿಮೆ ಉತ್ಸಾಹ** - ವೇಗವಾದ ಉತ್ತರಗಳಿಗೆ GPT-5 ಮಾದರಿ.

**ಬಹುಮಟ್ಟಿನ ಸಂವಾದ** - ವಿನಿಮಯಗಳ ನಡುವೆ ಕಾಂಟೆಕ್ಸ್ಟ್ ಅನ್ನು ಕಾಪಾಡುವುದು.

**ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್** - ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳ ಮೂಲಕ ಮಾಡೆಲ್ ಭಾವಚಿತ್ರವನ್ನು ಹೊಂದಿಸುವುದು.

**ಸ್ವ-ಪರಿಶೀಲನೆ** - ಮಾಡೆಲ್ ತನ್ನ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಅಳೆಯುತ್ತದೆ ಮತ್ತು ಸುಧಾರಿಸುತ್ತದೆ.

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ನಿಯಮಿತ ಮೌಲ್ಯಮಾಪನ ಚಟುವಟಿಕೆ.

**ಕಾರ್ಯ ನಿರ್ವಹಣಾ ಮಾದರಿ** - ಯೋಜನೆ → ನಿರ್ವಹಣೆ → ಸಾರಾಂಶ.

## RAG (ರಿಟ್ರೀವಲ್-ಆಗ್ಮೆಂಟೆಡ್ ಜನರೇಶನ್) - [ಮೊಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ದಸ್ತಾವೇಜು ಪ್ರಕ್ರಿಯೆ ಪೈಪ್‌ಲೈನ್** - ಲೋಡ್ → ಚುಂಕ್ → ಎಂಬೆಡ್ → ಸಂಗ್ರಹ.

**ಇನ್-ಮೆಮೊರಿ ಎಂಬೆಡ್ಡಿಂಗ್ ಸಂಗ್ರಹ** - ಪರೀಕ್ಷೆಗಾಗಿ ಅಸ್ಥಾಯಿ ಸಂಗ್ರಹಣೆ.

**RAG** - ಪ್ರತಿಕ್ರಿಯೆಗಳಿಗೆ ನೆಲಗಾರಿಕೆಗಾಗಿ ರಿಟ್ರೀವಲ್ ಮತ್ತು ಜನರೇಶನ್ ಸಂಯೋಜನೆ.

**ಸಮಾನತೆ ಅಂಕಿ** - ಅರ್ಥಾತ್ಮಕ ಸಮಾನತೆಯ ಮಾನ (0-1).

**ಮೂಲ ಉಲ್ಲೇಖ** - ಪಡೆದ ವಿಷಯದ ಮೆಟಾಡೇಟಾ.

## ಏಜೆಂಟ್ಗಳು ಮತ್ತು ಉಪಕರಣಗಳು - [ಮೊಡ್ಯೂಲ್ 04](../04-tools/README.md)

**@Tool ಅನ್‌ನೋಟೇಶನ್** - ಜಾವಾ ವಿಧಾನಗಳನ್ನು ಎಐ ಕರೆಮಾಡಬಹುದಾದ ಉಪಕರಣಗಳಾಗಿ ಗುರುತಿಸುತ್ತದೆ.

**ರಿಯಾಕ್ಟ್ ಪ್ಯಾಟರ್ನ್** - ಯುಕ್ತಿವಾದ → ಕ್ರಿಯೆ → ಗಮನ → ಪುನರಾವರ್ತನೆ.

**ಸೆಷನ್ ನಿರ್ವಹಣೆ** - ವಿವಿಧ ಬಳಕೆದಾರರಿಗೆ ವಿಭಿನ್ನ ಕಾಂಟೆಕ್ಸ್ಟ್‌ಗಳು.

**ಟೂಲ್** - ಎಐ ಏಜೆಂಟ್ ಕರೆಮಾಡಬಹುದಾದ ಕಾರ್ಯ.

**ಟೂಲ್ ವಿವರಣೆ** - ಉಪಕರಣದ ಉದ್ದೇಶ ಮತ್ತು ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ದಾಖಲೆ.

## ಏಜೆಂಟಿಕ್ ಮೊಡ್ಯೂಲ್ - [ಮೊಡ್ಯೂಲ್ 05](../05-mcp/README.md)

**@Agent ಅನ್‌ನೋಟೇಶನ್** - ಇಂಟರ್ಫೇಸ್‌ಗಳನ್ನು ಏಜೆಂಟ್‌ಗಳಾಗಿ ಘೋಷಿಸುವುದು, ಪ್ರಕಟಣಾತ್ಮಕ ವರ್ತನೆ ವ್ಯಾಖ್ಯಾನ.

**ಏಜೆಂಟ್ ಲಿಸನರ್** - `beforeAgentInvocation()` ಮತ್ತು `afterAgentInvocation()` ಮೂಲಕ ಏಜೆಂಟ್ ಕಾರ್ಯ ಚಟುವಟಿಕೆಗಳನ್ನ ತಗಲಿಸುವ ಹೂಕ್.

**ಏಜೆಂಟಿಕ್ స్కೋಪ್** - ಏಜೆಂಟ್‌ಗಳು ಫಲಿತಾಂಶಗಳನ್ನು `outputKey` ಬಳಸಿ ಸಂಗ್ರಹಿಸುವ ಹಂಚಿಕೆಯಾಗುವ ಮೆಮೊರಿ; ಡೌನ್‌ಸ್ಟ್ರೀಮ್ ಏಜೆಂಟ್‌ಗಳಿಗೆ ನೀಡಲು.

**AgenticServices** - `agentBuilder()` ಮತ್ತು `supervisorBuilder()` ಬಳಸಿ ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವ ಫ್ಯಾಕ್ಟರಿ.

**ಶರತವರ ಸುತ್ತಾಟ ಕಾರ್ಯವಳಿ** - ಶರತ್ತುಗಳ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ತಜ್ಞ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಮಾರ್ಗನಿರ್ದೇಶ.

**ಮನುಷ್ಯ-ಇನ್-ದಿ-ಲುಪ್** - ಮನುಷ್ಯ ಇನ್ಪುಟ್ ಅಥವಾ ವಿಷಯ ಪರಿಶೀಲನೆಗಾಗಿ ಕಾರ್ಯವಳಿ ಮಾದರಿ.

**langchain4j-agentic** - ಪ್ರಕಟಣಾತ್ಮಕ ಏಜೆಂಟ್ ನಿರ್ಮಾಣಕ್ಕಾಗಿ ಸಾಂಪ್ರದಾಯಕ ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ನಿರ್ಭರತೆ (ಪ್ರಾಯೋಗಿಕ).

**ಲೂಪ್ ಕಾರ್ಯವಳಿ** - ಗುಣಮಟ್ಟ ಅಂಕಿಅಂಶ ≥ 0.8 ಮುಂತಾದ ಶರತ್ತು ತಾಕಲು ಏಜೆಂಟ್ ಕಾರ್ಯವನ್ನು ಪುನರಾವರ್ತಿಸುವುದು.

**outputKey** - ಏಜೆಂಟಿಕ್ స్కೋಪ್‌ನಲ್ಲಿ ಫಲಿತಾಂಶಗಳು ಸಂಗ್ರಹಿಸುವ ಸ್ಥಳ ಸೂಚಿಸುವ ಏಜೆಂಟ್ ಅನ್‌ನೋಟೇಶನ್ ಪ್ಯಾರಾಮೀಟರ್.

**ಪ್ಯಾರಲಲ್ ಕಾರ್ಯವಳಿ** - ಸ್ವತಂತ್ರ ಕಾರ್ಯಗಳಿಗೆ ಬಹು ಏಜೆಂಟ್‌ಗಳನ್ನು ಸಮಕಾಲೀನವಾಗಿ ಕಾರ್ಯಗತಗೊಳಿಸುವುದು.

**ಪ್ರತಿಕ್ರಿಯೆ ತಂತ್ರ** - ಮೇಲ್ವಿಚಾರಕ ಅಂತಿಮ ಉತ್ತರವನ್ನು ರೂಪಿಸುವ ವಿಧಾನ: LAST, SUMMARY, ಅಥವಾ SCORED.

**ಕ್ರಮಶಃ ಕಾರ್ಯವಳಿ** - ಔಟ್‌ಪುಟ್ ಮುಂದಿನ ಹಂತಕ್ಕೆ ಹರಿವಂತೆ ಏಜೆಂಟ್‌ಗಳನ್ನು ಕ್ರಮಬದ್ಧವಾಗಿ ನಿರ್ವಹಣೆ.

**ಮೇಲ್ವಿಚಾರಕ ಏಜೆಂಟ್ ಮಾದರಿ** - ಸಬ್ಏಜೆಂಟ್‌ಗಳ ಏರ್ಪಾಡು ನಿರ್ಧಾರವನ್ನು ಡೈನಾಮಿಕ್‌ವಾಗಿ ಮಾಡುವ LLM ಮೇಲುನೋಡಣಾ ಏಜೆಂಟ್ ಮಾದರಿ.

## ಮಾಡೆಲ್_Context ಪ್ರೋಟೋಕಾಲ್ (MCP) - [ಮೊಡ್ಯೂಲ್ 05](../05-mcp/README.md)

**langchain4j-mcp** - ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ™ನಲ್ಲಿ MCP ಸಂಯೋಜನೆಗಾಗಿ ಅವಲಂಬನೆ.

**MCP** - ಮಾಡೆಲ್_Context ಪ್ರೋಟೋಕಾಲ್: ಎಐ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಗಿನ ಉಪಕರಣಗಳ ಜೊತೆಗೆ ಸಂಪರ್ಕಿಸುವ ಪ್ರಮಾಣ.

**MCP ಕ್ಲೈಂಟ್** - MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಿಸಿ ಉಪಕರಣಗಳನ್ನು ಹುಡುಕುವ ಮತ್ತು ಉಪಯೋಗಿಸುವ ಅಪ್ಲಿಕೇಶನ್.

**MCP ಸರ್ವರ್** - ಎಐ ಸೇವೆಗೆ ಸ್ಪಷ್ಟ ವಿವರಣೆ ಮತ್ತು ಪ್ಯಾರಾಮೀಟರ್ ಸ್ಕೀಮಾ ಜೊತೆ MCP ಮೂಲಕ ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುವ ಸೇವೆ.

**McpToolProvider** - ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಘಟಕ, AI ಸೇವೆಗಳು ಮತ್ತು ಏಜೆಂಟ್‌ಗಳಲ್ಲಿ ಬಳಕೆಗಾಗಿ MCP ಉಪಕರಣಗಳನ್ನು ಮುಚ್ಚುವವನು.

**McpTransport** - MCP ಸಂವಹನಕ್ಕಾಗಿ ಇಂಟರ್ಫೇಸ್. ಅಳವಡಿಕೆಗಳು: ಸ್ಟಿಡಿಯೋ ಮತ್ತು HTTP.

**ಸ್ಟಿಡಿಯೋ ಸಂವಹನ** - ಸ್ಥಳೀಯ ಪ್ರಕ್ರಿಯೆ ಸಂವಹನ stdin/stdout ಮುಖಾಂತರ. ಫೈಲ್ ಸಿಸ್ಟಮ್ ಪ್ರವೇಶ ಅಥವಾ ಕಮಾಂಡ್ ಲೈನ್ ಉಪಕರಣಗಳಿಗೆ ಉಪಯುಕ್ತ.

**StdioMcpTransport** - ಸಬ್‌ಪ್ರೊಸೆಸ್ನಾಗಿ MCP ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸುವ ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಅನುಷ್ಠಾನ.

**ಅಪಕರಣ ಕಂಡುಹಿಡಿಯುವಿಕೆ** - ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳಿಗೆ ವಿವರಣೆ ಮತ್ತು ಸ್ಕೀಮಾ ಧಾರಣೆೊಂದಿಗೆ ಕ್ಲೈಂಟ್ ಪ್ರಶ್ನೆ.
