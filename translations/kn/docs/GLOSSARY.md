## ಮೂಲಭೂತ ಕಲ್ಪನೆಗಳು

**AI ಏಜೆಂಟ್** - ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಯುಕ್ತಿವಾದ ಮತ್ತು ಕ್ರಿಯೆ ಮಾಡುವ AI ಬಳಕೆದಾರ ವ್ಯವಸ್ಥೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಚೈನ್** - ಕ್ರಮವಾಗಿ ಕಾರ್ಯಾಚರಣೆಗಳ ಸರಣಿ, ಇಲ್ಲಿ ಔಟ್‌ಪುಟ್ ಮುಂದಿನ ಹಂತಕ್ಕೆ ನೀಡಲಾಗುತ್ತದೆ.

**ಚಂಕಿಂಗ್** - ದಾಖಲೆಗಳನ್ನು ಸಣ್ಣ ತುಂಡುಗಳಾಗಿ ವಿಭಜಿಸುವುದು. ಸಾಮಾನ್ಯ: 300-500 ಟೋಕನ್ಗಳು, ಅತಿರಿಕ್ತ ಭಾಗದೊಂದಿಗೆ. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಸಂದರ್ಭ ವಿಂಡೋ** - ಮಾದರಿ ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದಾದ ಗರಿಷ್ಠ ಟೋಕನ್ಗಳು. GPT-5: 400K ಟೋಕನ್ಗಳು.

**ಎಂಬೆಡ್ಡಿಂಗ್ಸ್** - ಪಠ್ಯದ ಅರ್ಥವನ್ನು ಪ್ರತಿನಿಧಿಸುವ ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್‌ಗಳು. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್** - ಮಾದರಿ ಬಾಹ್ಯ ಕಾರ್ಯಗಳನ್ನು ಕರೆ ಮಾಡಲು ರಚನಾತ್ಮಕ ವಿನಂತಿಗಳನ್ನು ರಚಿಸುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಹ್ಯಾಲುಸಿನೇಷನ್** - ಮಾದರಿಗಳು ತಪ್ಪು ಆದರೆ ಸಾಧ್ಯವಿರುವ ಮಾಹಿತಿಯನ್ನು ರಚಿಸುವಾಗ.

**ಪ್ರಾಂಪ್ಟ್** - ಭಾಷಾ ಮಾದರಿಗೆ ಪಠ್ಯ ಇನ್ಪುಟ್. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಸಾಮಾಂತಾರ ಹುಡುಕಾಟ** - ಕೀವರ್ಡ್‌ಗಳ ಬದಲು ಅರ್ಥದ ಮೂಲಕ ಹುಡುಕಾಟ, ಎಂಬೆಡ್ಡಿಂಗ್ಸ್ ಬಳಸಿ. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಸ್ಟೇಟ್‌ಫುಲ್ ವಿರುದ್ಧ ಸ್ಟೇಟ್‌ಲೆಸ್** - ಸ್ಟೇಟ್‌ಲೆಸ್: ಸ್ಮರಣೆ ಇಲ್ಲ. ಸ್ಟೇಟ್‌ಫುಲ್: ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಪಾಡುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೋಕನ್ಗಳು** - ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಪಠ್ಯ ಘಟಕಗಳು. ವೆಚ್ಚ ಮತ್ತು ಮಿತಿಗಳನ್ನು ಪ್ರಭಾವಿಸುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೂಲ್ ಚೈನಿಂಗ್** - ಕ್ರಮವಾಗಿ ಉಪಕರಣಗಳ ಕಾರ್ಯಾಚರಣೆ, ಔಟ್‌ಪುಟ್ ಮುಂದಿನ ಕರೆಗಾಗಿ ಮಾಹಿತಿ ನೀಡುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

## LangChain4j ಘಟಕಗಳು

**AiServices** - ಪ್ರಕಾರ-ಸುರಕ್ಷಿತ AI ಸೇವಾ ಇಂಟರ್ಫೇಸ್ಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

**OpenAiOfficialChatModel** - OpenAI ಮತ್ತು ಅಜೂರ್ OpenAI ಮಾದರಿಗಳಿಗಾಗಿ ಏಕೀಕೃತ ಕ್ಲೈಂಟ್.

**OpenAiOfficialEmbeddingModel** - OpenAI ಅಧಿಕೃತ ಕ್ಲೈಂಟ್ ಬಳಸಿ ಎಂಬೆಡ್ಡಿಂಗ್ಸ್ ರಚಿಸುತ್ತದೆ (OpenAI ಮತ್ತು ಅಜೂರ್ OpenAI ಎರಡನ್ನೂ ಬೆಂಬಲಿಸುತ್ತದೆ).

**ChatModel** - ಭಾಷಾ ಮಾದರಿಗಳ ಮೂಲ ಇಂಟರ್ಫೇಸ್.

**ChatMemory** - ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಪಾಡುತ್ತದೆ.

**ContentRetriever** - RAG ಗಾಗಿ ಸಂಬಂಧಿತ ದಾಖಲೆ ಚಂಕ್‌ಗಳನ್ನು ಹುಡುಕುತ್ತದೆ.

**DocumentSplitter** - ದಾಖಲೆಗಳನ್ನು ಚಂಕ್‌ಗಳಾಗಿ ವಿಭಜಿಸುತ್ತದೆ.

**EmbeddingModel** - ಪಠ್ಯವನ್ನು ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್‌ಗಳಾಗಿ ಪರಿವರ್ತಿಸುತ್ತದೆ.

**EmbeddingStore** - ಎಂಬೆಡ್ಡಿಂಗ್ಸ್ ಸಂಗ್ರಹಿಸುತ್ತದೆ ಮತ್ತು ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

**MessageWindowChatMemory** - ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ಕಾಪಾಡುತ್ತದೆ.

**PromptTemplate** - `{{variable}}` ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್‌ಗಳೊಂದಿಗೆ ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

**TextSegment** - ಮೆಟಾಡೇಟಾ ಹೊಂದಿರುವ ಪಠ್ಯ ಚಂಕ್. RAG ನಲ್ಲಿ ಬಳಸಲಾಗುತ್ತದೆ.

**ToolExecutionRequest** - ಉಪಕರಣ ಕಾರ್ಯಾಚರಣೆಯ ವಿನಂತಿಯನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತದೆ.

**UserMessage / AiMessage / SystemMessage** - ಸಂಭಾಷಣಾ ಸಂದೇಶ ಪ್ರಕಾರಗಳು.

## AI/ML ಕಲ್ಪನೆಗಳು

**ಫ್ಯೂ-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ ಉದಾಹರಣೆಗಳನ್ನು ನೀಡುವುದು. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾದರಿ (LLM)** - ವಿಶಾಲ ಪಠ್ಯ ಡೇಟಾದ ಮೇಲೆ ತರಬೇತಿಗೊಂಡ AI ಮಾದರಿಗಳು.

**ಯುಕ್ತಿವಾದ ಪ್ರಯತ್ನ** - ಯೋಚನೆಯ ಆಳವನ್ನು ನಿಯಂತ್ರಿಸುವ GPT-5 ಪರಿಮಾಣ. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ತಾಪಮಾನ** - ಔಟ್‌ಪುಟ್ ಅಸ್ಪಷ್ಟತೆಯನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ. ಕಡಿಮೆ=ನಿರ್ಧಿಷ್ಟ, ಹೆಚ್ಚು=ಸೃಜನಶೀಲ.

**ವೆಕ್ಟರ್ ಡೇಟಾಬೇಸ್** - ಎಂಬೆಡ್ಡಿಂಗ್ಸ್‌ಗಾಗಿ ವಿಶೇಷ ಡೇಟಾಬೇಸ್. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಜೀરો-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ - [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಚೈನ್-ಆಫ್-ಥಾಟ್** - ಉತ್ತಮ ನಿಖರತೆಗಾಗಿ ಹಂತ ಹಂತವಾಗಿ ಯುಕ್ತಿವಾದ.

**ನಿಯಂತ್ರಿತ ಔಟ್‌ಪುಟ್** - ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪ ಅಥವಾ ರಚನೆಯನ್ನು ಜಾರಿಗೊಳಿಸುವುದು.

**ಹೆಚ್ಚು ಉತ್ಸಾಹ** - ಸಮಗ್ರ ಯುಕ್ತಿವಾದಕ್ಕಾಗಿ GPT-5 ಮಾದರಿ.

**ಕಡಿಮೆ ಉತ್ಸಾಹ** - ತ್ವರಿತ ಉತ್ತರಗಳಿಗಾಗಿ GPT-5 ಮಾದರಿ.

**ಬಹು-ತಿರುವು ಸಂಭಾಷಣೆ** - ವಿನಿಮಯಗಳ ನಡುವೆ ಸಾಂದರ್ಭಿಕತೆಯನ್ನು ಕಾಪಾಡುವುದು.

**ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್** - ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳ ಮೂಲಕ ಮಾದರಿ ವ್ಯಕ್ತಿತ್ವವನ್ನು ಹೊಂದಿಸುವುದು.

**ಸ್ವ-ಪರಿಶೀಲನೆ** - ಮಾದರಿ ತನ್ನ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸುತ್ತದೆ.

**ರಚನಾತ್ಮಕ ವಿಶ್ಲೇಷಣೆ** - ನಿಶ್ಚಿತ ಮೌಲ್ಯಮಾಪನ ಚಟುವಟಿಕೆ.

**ಕಾರ್ಯ ನಿರ್ವಹಣಾ ಮಾದರಿ** - ಯೋಜನೆ → ಕಾರ್ಯಗತಗೊಳಿಸಿ → ಸಾರಾಂಶ.

## RAG (ಪುನಃಪ್ರಾಪ್ತಿ-ವೃದ್ಧಿತ ತಯಾರಿಕೆ) - [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ದಾಖಲೆ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್ಲೈನ್** - ಲೋಡ್ → ಚಂಕ್ → ಎಂಬೆಡ್ → ಸಂಗ್ರಹಿಸಿ.

**ಇನ್-ಮೆಮೊರಿ ಎಂಬೆಡ್ಡಿಂಗ್ ಸ್ಟೋರ್** - ಪರೀಕ್ಷೆಗಾಗಿ ಅಸ್ಥಿರ ಸಂಗ್ರಹಣೆ.

**RAG** - ಪ್ರತಿಕ್ರಿಯೆಗಳಿಗೆ ನೆಲೆಸಲು ಪುನಃಪ್ರಾಪ್ತಿಯನ್ನು ತಯಾರಿಕೆಗೆ ಸಂಯೋಜಿಸುತ್ತದೆ.

**ಸಮಾನತೆ ಅಂಕ** - ಅರ್ಥಸಾಮ್ಯತೆಯ (0-1) ಅಳತೆ.

**ಮೂಲ ಉಲ್ಲೇಖ** - ಪುನಃಪ್ರಾಪ್ತಿಯಾದ ವಿಷಯದ ಮೆಟಾಡೇಟಾ.

## ಏಜೆಂಟ್ಗಳು ಮತ್ತು ಉಪಕರಣಗಳು - [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**@Tool ಅ್ಯಾನೋಟೇಶನ್** - ಜಾವಾ ವಿಧಾನಗಳನ್ನು AI-ಕರೆ ಮಾಡಬಹುದಾದ ಉಪಕರಣಗಳಾಗಿ ಗುರುತಿಸುತ್ತದೆ.

**ReAct ಮಾದರಿ** - ಯುಕ್ತಿವಾದ → ಕ್ರಿಯೆ → ಗಮನಿಸಿ → ಪುನರಾವರ್ತನೆ.

**ಸೆಷನ್ ನಿರ್ವಹಣೆ** - ವಿಭಿನ್ನ ಬಳಕೆದಾರರಿಗಾಗಿ ಪ್ರತ್ಯೇಕ ಸಾಂದರ್ಭಿಕತೆಗಳು.

**ಉಪಕರಣ** - AI ಏಜೆಂಟ್ ಕರೆ ಮಾಡಬಹುದಾದ ಕಾರ್ಯ.

**ಉಪಕರಣ ವಿವರಣೆ** - ಉಪಕರಣದ ಉದ್ದೇಶ ಮತ್ತು ಪರಿಮಾಣಗಳ ಡಾಕ್ಯುಮೆಂಟೇಶನ್.

## ಮಾದರಿ ಸಾಂದರ್ಭಿಕ ಪ್ರೋಟೋಕಾಲ್ (MCP) - [ಮಾಡ್ಯೂಲ್ 05](../05-mcp/README.md)

**ಡೋಕರ್ ಸಾರಿಗೆ** - ಡೋಕರ್ ಕಂಟೈನರ್‌ನಲ್ಲಿ MCP ಸರ್ವರ್.

**MCP** - AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಬಾಹ್ಯ ಉಪಕರಣಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನಕ.

**MCP ಕ್ಲೈಂಟ್** - MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಅಪ್ಲಿಕೇಶನ್.

**MCP ಸರ್ವರ್** - MCP ಮೂಲಕ ಉಪಕರಣಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸುವ ಸೇವೆ.

**ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್ (SSE)** - HTTP ಮೂಲಕ ಸರ್ವರ್-ಟು-ಕ್ಲೈಂಟ್ ಸ್ಟ್ರೀಮಿಂಗ್.

**ಸ್ಟ್ಡಿಯೋ ಸಾರಿಗೆ** - stdin/stdout ಮೂಲಕ ಉಪಸರ್ವರ್ ಆಗಿ ಸರ್ವರ್.

**ಸ್ಟ್ರೀಮಬಲ್ HTTP ಸಾರಿಗೆ** - ನೈಜಕಾಲ ಸಂವಹನಕ್ಕಾಗಿ SSE ಹೊಂದಿರುವ HTTP.

**ಉಪಕರಣ ಪತ್ತೆ** - ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳಿಗಾಗಿ ಕ್ಲೈಂಟ್ ಸರ್ವರ್ ಅನ್ನು ಪ್ರಶ್ನಿಸುತ್ತದೆ.

## ಅಜೂರ್ ಸೇವೆಗಳು - [ಮಾಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಅಜೂರ್ AI ಹುಡುಕಾಟ** - ವೆಕ್ಟರ್ ಸಾಮರ್ಥ್ಯಗಳೊಂದಿಗೆ ಕ್ಲೌಡ್ ಹುಡುಕಾಟ. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಅಜೂರ್ ಡೆವಲಪರ್ CLI (azd)** - ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಯೋಜಿಸುತ್ತದೆ.

**ಅಜೂರ್ OpenAI** - ಮೈಕ್ರೋಸಾಫ್ಟ್‌ನ ಎಂಟರ್‌ಪ್ರೈಸ್ AI ಸೇವೆ.

**ಬೈಸೆಪ್** - ಅಜೂರ್ ಮೂಲಸೌಕರ್ಯ-ಆಸ್-ಕೋಡ್ ಭಾಷೆ. [ಮೂಲಸೌಕರ್ಯ ಮಾರ್ಗದರ್ಶಿ](../01-introduction/infra/README.md)

**ನಿಯೋಜನೆ ಹೆಸರು** - ಅಜೂರ್‌ನಲ್ಲಿ ಮಾದರಿ ನಿಯೋಜನೆಗೆ ಹೆಸರು.

**GPT-5** - ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ಇತ್ತೀಚಿನ OpenAI ಮಾದರಿ. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

## ಪರೀಕ್ಷೆ ಮತ್ತು ಅಭಿವೃದ್ಧಿ - [ಪರೀಕ್ಷೆ ಮಾರ್ಗದರ್ಶಿ](TESTING.md)
