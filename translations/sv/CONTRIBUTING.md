# Contributing

Tack för ditt intresse att bidra! Vi välkomnar pull requests och förbättringar till denna kurs. Vänligen följ dessa riktlinjer när du skickar in ändringar:

1. **Inga hemligheter i commits.** Lämna aldrig in API-nycklar eller något konfidentiellt material. Använd en lokal `.env` för din egen testning.  
2. **Håll ändringarna fokuserade.** Små, avgränsade pull requests är lättare att granska och underhålla. Om du åtgärdar flera problem, vänligen skicka in dem separat.  
3. **Lägg till tester där det är möjligt.** Enhets- eller integrationstester hjälper till att fånga regressionsfel. För Spring Boot-moduler, inkludera tester under mappen `src/test/java`.  
4. **Följ kodstilskonventioner.** Använd den befintliga projektstrukturen som vägledning. Vi förlitar oss på Maven för att hantera beroenden och Spring Boot Maven-plugin för att paketera applikationer.

Tack för att du hjälper till att göra denna kurs bättre!
