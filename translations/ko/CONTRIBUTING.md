# Contributing

기여해 주셔서 감사합니다! 이 강좌에 대한 풀 리퀘스트와 개선 사항을 환영합니다. 변경 사항을 제출할 때 다음 지침을 따라 주세요:

1. **커밋에 비밀 정보를 포함하지 마세요.** API 키나 기밀 자료를 절대 커밋하지 마세요. 개인 테스트용으로는 로컬 `.env` 파일을 사용하세요.
2. **변경 사항을 집중적으로 유지하세요.** 작고 범위가 명확한 풀 리퀘스트가 검토 및 유지 관리하기 더 쉽습니다. 여러 문제를 수정하는 경우에는 각각 별도로 제출해 주세요.
3. **가능한 경우 테스트를 추가하세요.** 단위 테스트나 통합 테스트는 회귀를 방지하는 데 도움이 됩니다. Spring Boot 모듈의 경우 `src/test/java` 폴더 아래에 테스트를 포함하세요.
4. **코드 스타일 규칙을 따르세요.** 기존 프로젝트 구조를 참고하세요. 의존성 관리는 Maven에 맡기고, Spring Boot Maven 플러그인을 사용해 애플리케이션을 패키징합니다.

이 강좌를 더 좋게 만드는 데 도움을 주셔서 감사합니다!
