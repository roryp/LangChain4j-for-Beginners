## Основні поняття

**ШІ-агент** - Система, яка використовує штучний інтелект для міркування та автономних дій. [Модуль 04](../04-tools/README.md)

**Ланцюжок** - Послідовність операцій, де вихід служить вхідним для наступного кроку.

**Розбиття на частини (Chunking)** - Розділення документів на менші фрагменти. Типово: 300-500 токенів з перекриттям. [Модуль 03](../03-rag/README.md)

**Вікно контексту** - Максимальна кількість токенів, які модель може обробити. GPT-5: 400K токенів.

**Ембеддінги** - Числові вектори, що представляють значення тексту. [Модуль 03](../03-rag/README.md)

**Виклик функцій** - Модель генерує структуровані запити для виклику зовнішніх функцій. [Модуль 04](../04-tools/README.md)

**Галюцинація** - Коли моделі генерують неправильну, але правдоподібну інформацію.

**Промпт** - Текстовий вхід у мовну модель. [Модуль 02](../02-prompt-engineering/README.md)

**Семантичний пошук** - Пошук за значенням за допомогою ембеддінгів, а не ключових слів. [Модуль 03](../03-rag/README.md)

**Становий (Stateful) проти безстанного (Stateless)** - Безстанний: немає пам'яті. Становий: зберігає історію розмови. [Модуль 01](../01-introduction/README.md)

**Токени** - Базові одиниці тексту, які обробляють моделі. Впливають на витрати та обмеження. [Модуль 01](../01-introduction/README.md)

**Ланцюжок викликів інструментів** - Послідовне виконання інструментів, де вихід інформує наступний виклик. [Модуль 04](../04-tools/README.md)

## Компоненти LangChain4j

**AiServices** - Створює типобезпечні інтерфейси сервісів ШІ.

**OpenAiOfficialChatModel** - Уніфікований клієнт для моделей OpenAI та Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - Створює ембеддінги за допомогою офіційного клієнта OpenAI (підтримує як OpenAI, так і Azure OpenAI).

**ChatModel** - Основний інтерфейс для мовних моделей.

**ChatMemory** - Підтримує історію розмови.

**ContentRetriever** - Знаходить релевантні фрагменти документів для RAG.

**DocumentSplitter** - Розбиває документи на частини.

**EmbeddingModel** - Конвертує текст у числові вектори.

**EmbeddingStore** - Зберігає та отримує ембеддінги.

**MessageWindowChatMemory** - Підтримує ковзаюче вікно останніх повідомлень.

**PromptTemplate** - Створює повторно використовувані промпти з плейсхолдерами `{{variable}}`.

**TextSegment** - Текстовий фрагмент з метаданими. Використовується в RAG.

**ToolExecutionRequest** - Представляє запит на виконання інструменту.

**UserMessage / AiMessage / SystemMessage** - Типи повідомлень у розмові.

## Поняття ШІ/МЛ

**Навчання з кількома прикладами (Few-Shot Learning)** - Надання прикладів у промптах. [Модуль 02](../02-prompt-engineering/README.md)

**Великі мовні моделі (LLM)** - Моделі ШІ, натреновані на великій кількості текстових даних.

**Зусилля міркування (Reasoning Effort)** - Параметр GPT-5, що контролює глибину мислення. [Модуль 02](../02-prompt-engineering/README.md)

**Температура** - Контролює випадковість виходу. Низька = детермінований, висока = креативний.

**Векторна база даних** - Спеціалізована база даних для ембеддінгів. [Модуль 03](../03-rag/README.md)

**Навчання без прикладів (Zero-Shot Learning)** - Виконання задач без прикладів. [Модуль 02](../02-prompt-engineering/README.md)

## Інженерія промптів - [Модуль 02](../02-prompt-engineering/README.md)

**Ланцюг думок (Chain-of-Thought)** - Покрокове міркування для кращої точності.

**Обмежений вихід (Constrained Output)** - Примус до конкретного формату або структури.

**Висока схильність (High Eagerness)** - Шаблон GPT-5 для ретельного міркування.

**Низька схильність (Low Eagerness)** - Шаблон GPT-5 для швидких відповідей.

**Багатотурова розмова** - Підтримка контексту між обмінами.

**Ролеве підказування** - Встановлення персонажу моделі через системні повідомлення.

**Саморефлексія** - Модель оцінює та покращує власний вихід.

**Структурований аналіз** - Фіксована рамка оцінювання.

**Схема виконання завдання** - План → Виконання → Підсумок.

## RAG (Retrieval-Augmented Generation) - [Модуль 03](../03-rag/README.md)

**Конвеєр обробки документів** - Завантажити → розбити → векторизувати → зберегти.

**Сховище ембеддінгів в пам'яті** - Непостійне сховище для тестування.

**RAG** - Поєднує витяг інформації з генерацією для прив'язки відповідей до джерел.

**Оцінка схожості** - Міра (0-1) семантичної подібності.

**Посилання на джерело** - Метадані про витягнутий контент.

## Агенти та інструменти - [Модуль 04](../04-tools/README.md)

**@Tool Annotation** - Позначає Java-методи як доступні для виклику ШІ.

**ReAct Pattern** - Міркувати → Діяти → Спостерігати → Повторювати.

**Управління сесією** - Окремі контексти для різних користувачів.

**Інструмент (Tool)** - Функція, яку може викликати ШІ-агент.

**Опис інструменту** - Документація про призначення та параметри інструменту.

## Протокол контексту моделі (MCP) - [Модуль 05](../05-mcp/README.md)

**MCP** - Стандарт для підключення ШІ-застосунків до зовнішніх інструментів.

**MCP Client** - Застосунок, що підключається до MCP-серверів.

**MCP Server** - Сервіс, який експонує інструменти через MCP.

**Stdio Transport** - Сервер як підпроцес через stdin/stdout.

**Виявлення інструментів** - Клієнт запитує сервер про доступні інструменти.

## Сервіси Azure - [Модуль 01](../01-introduction/README.md)

**Azure AI Search** - Хмарний пошук з підтримкою векторів. [Модуль 03](../03-rag/README.md)

**Azure Developer CLI (azd)** - Розгортає ресурси Azure.

**Azure OpenAI** - Корпоративний сервіс ШІ від Microsoft.

**Bicep** - Мова інфраструктури як коду для Azure. [Посібник з інфраструктури](../01-introduction/infra/README.md)

**Назва розгортання** - Ім'я для розгортання моделі в Azure.

**GPT-5** - Остання модель OpenAI з контролем міркування. [Модуль 02](../02-prompt-engineering/README.md)

## Тестування та розробка - [Посібник з тестування](TESTING.md)

**Dev Container** - Контейнеризоване середовище розробки. [Конфігурація](../../../.devcontainer/devcontainer.json)

**Mockito** - Фреймворк для мокування в Java.

**Spring Boot** - Фреймворк для Java-застосунків. [Модуль 01](../01-introduction/README.md)
