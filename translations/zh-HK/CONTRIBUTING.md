# Contributing

感謝您有興趣作出貢獻！我們歡迎拉取請求和對本課程的改進。提交更改時，請遵循以下指引：

1. **提交中不得包含秘密。** 絕不提交 API 金鑰或任何機密資料。請使用本地 `.env` 進行個人測試。
2. **保持更改專注。** 小而具範圍的拉取請求更易於審查和維護。如果您修復多個問題，請分別提交。
3. **盡可能添加測試。** 單元測試或整合測試有助於捕捉回歸。對於 Spring Boot 模組，請在 `src/test/java` 資料夾下包含測試。
4. **遵循程式碼風格慣例。** 使用現有專案結構作為指引。我們依賴 Maven 管理依賴，並使用 Spring Boot Maven 插件打包應用程式。

感謝您幫助改進本課程！
