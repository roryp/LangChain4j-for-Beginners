# Bijdragen

Bedankt voor je interesse om bij te dragen! We verwelkomen pull requests en verbeteringen voor deze cursus. Volg deze richtlijnen bij het indienen van wijzigingen:

1. **Geen geheimen in commits.** Voeg nooit API-sleutels of vertrouwelijk materiaal toe in commits. Gebruik een lokale `.env` voor je eigen tests.
2. **Houd wijzigingen gefocust.** Kleine, afgebakende pull requests zijn makkelijker te beoordelen en te onderhouden. Als je meerdere problemen oplost, dien ze dan apart in.
3. **Voeg waar mogelijk tests toe.** Unit- of integratietests helpen regressies te voorkomen. Voor Spring Boot-modules, voeg tests toe onder de map `src/test/java`.
4. **Volg de code stijl conventies.** Gebruik de bestaande projectstructuur als leidraad. We vertrouwen op Maven voor het beheren van afhankelijkheden en de Spring Boot Maven plugin voor het verpakken van applicaties.

Bedankt dat je helpt deze cursus beter te maken!
