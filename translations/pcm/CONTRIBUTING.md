# Contributing

Thank you for your interest for contribute! We dey welcome pull
requests and improvements to dis course. Abeg follow these
guidelines wen you dey submit changes:

1. **No secrets for commits.** No ever commit API keys or any
   confidential material. Use local `.env` for your own testing.
2. **Keep changes focused.** Small, scoped pull requests dey easier
   to review and maintain. If you dey fix multiple issues, abeg
   submit dem separately.
3. **Add tests where possible.** Unit or integration tests go help
   catch regressions. For Spring Boot modules, put tests under
   the `src/test/java` folder.
4. **Follow code style conventions.** Use the existing project
   structure as guide. We dey rely on Maven to manage dependencies and
   the Spring Boot Maven plugin to package applications.

Thank you for helping make dis course beta!
