# Contribuer

Merci de votre intérêt pour contribuer ! Nous accueillons les pull requests et les améliorations de ce cours. Veuillez suivre ces directives lors de la soumission de modifications :

1. **Pas de secrets dans les commits.** Ne jamais commettre de clés API ou tout matériel confidentiel. Utilisez un `.env` local pour vos propres tests.
2. **Gardez les modifications ciblées.** Les pull requests petites et ciblées sont plus faciles à examiner et à maintenir. Si vous corrigez plusieurs problèmes, veuillez les soumettre séparément.
3. **Ajoutez des tests lorsque c’est possible.** Les tests unitaires ou d’intégration aident à détecter les régressions. Pour les modules Spring Boot, incluez des tests dans le dossier `src/test/java`.
4. **Respectez les conventions de style de code.** Utilisez la structure existante du projet comme guide. Nous comptons sur Maven pour gérer les dépendances et le plugin Spring Boot Maven pour empaqueter les applications.

Merci de contribuer à améliorer ce cours !
