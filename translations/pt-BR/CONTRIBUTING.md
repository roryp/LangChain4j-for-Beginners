# Contribuindo

Obrigado pelo seu interesse em contribuir! Recebemos pull requests e melhorias para este curso. Por favor, siga estas diretrizes ao enviar alterações:

1. **Sem segredos nos commits.** Nunca faça commit de chaves de API ou qualquer material confidencial. Use um `.env` local para seus próprios testes.
2. **Mantenha as mudanças focadas.** Pull requests pequenos e específicos são mais fáceis de revisar e manter. Se você estiver corrigindo vários problemas, envie-os separadamente.
3. **Adicione testes sempre que possível.** Testes unitários ou de integração ajudam a detectar regressões. Para módulos Spring Boot, inclua testes na pasta `src/test/java`.
4. **Siga as convenções de estilo de código.** Use a estrutura existente do projeto como guia. Confiamos no Maven para gerenciar dependências e no plugin Spring Boot Maven para empacotar aplicações.

Obrigado por ajudar a melhorar este curso!
