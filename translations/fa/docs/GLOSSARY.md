## مفاهیم اصلی

**عامل AI** - سیستمی که برای استدلال و عمل مستقل از AI استفاده می‌کند. [ماژول ۰۴](../04-tools/README.md)

**زنجیره** - دنباله‌ای از عملیات که خروجی به مرحله بعدی می‌رود.

**تقسیم‌بندی** - شکستن اسناد به قطعات کوچکتر. معمولاً: ۳۰۰-۵۰۰ توکن با همپوشانی. [ماژول ۰۳](../03-rag/README.md)

**پنجره زمینه** - حداکثر تعداد توکن‌هایی که مدل می‌تواند پردازش کند. GPT-5: ۴۰۰ هزار توکن.

**تعابیر** - بردارهای عددی نمایانگر معنی متن. [ماژول ۰۳](../03-rag/README.md)

**فراخوانی تابع** - مدل درخواست‌های ساختاریافته را برای فراخوانی توابع خارجی تولید می‌کند. [ماژول ۰۴](../04-tools/README.md)

**توهم** - وقتی مدل‌ها اطلاعات نادرست اما محتمل تولید می‌کنند.

**پرامپت** - ورودی متنی به مدل زبان. [ماژول ۰۲](../02-prompt-engineering/README.md)

**جستجوی معنایی** - جستجو بر اساس معنی با استفاده از تعابیر، نه کلیدواژه‌ها. [ماژول ۰۳](../03-rag/README.md)

**دارای حالت در مقابل بدون حالت** - بدون حالت: بدون حافظه. دارای حالت: تاریخچه گفتگو را حفظ می‌کند. [ماژول ۰۱](../01-introduction/README.md)

**توکن‌ها** - واحدهای پایه متن که مدل‌ها پردازش می‌کنند. بر هزینه‌ها و محدودیت‌ها تأثیر دارد. [ماژول ۰۱](../01-introduction/README.md)

**زنجیره‌سازی ابزار** - اجرای متوالی ابزارها که خروجی برای فراخوانی بعدی استفاده می‌شود. [ماژول ۰۴](../04-tools/README.md)

## اجزای LangChain4j

**AiServices** - رابط‌های خدمات AI نوع ایمن ایجاد می‌کند.

**OpenAiOfficialChatModel** - کلاینت یکپارچه برای مدل‌های OpenAI و Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - ایجاد تعابیر با استفاده از کلاینت رسمی OpenAI (پشتیبانی از OpenAI و Azure OpenAI).

**ChatModel** - رابط اصلی برای مدل‌های زبانی.

**ChatMemory** - تاریخچه گفتگو را حفظ می‌کند.

**ContentRetriever** - تکه‌های مستند مرتبط برای RAG را پیدا می‌کند.

**DocumentSplitter** - اسناد را به قطعات تقسیم می‌کند.

**EmbeddingModel** - متن را به بردارهای عددی تبدیل می‌کند.

**EmbeddingStore** - تعابیر را ذخیره و بازیابی می‌کند.

**MessageWindowChatMemory** - پنجره لغزنده پیام‌های اخیر را نگه می‌دارد.

**PromptTemplate** - پرامپت‌های قابل استفاده مجدد با جایگزین‌های `{{variable}}` ایجاد می‌کند.

**TextSegment** - قطعه متنی با فراداده. در RAG استفاده می‌شود.

**ToolExecutionRequest** - نمایانگر درخواست اجرای ابزار.

**UserMessage / AiMessage / SystemMessage** - انواع پیام‌های گفتگو.

## مفاهیم AI/ML

**یادگیری چند نمونه‌ای** - ارائه نمونه‌ها در پرامپت‌ها. [ماژول ۰۲](../02-prompt-engineering/README.md)

**مدل زبان بزرگ (LLM)** - مدل‌های AI آموزش دیده بر روی داده‌های متنی وسیع.

**تلاش استدلال** - پارامتر GPT-5 کنترل‌کننده عمق تفکر. [ماژول ۰۲](../02-prompt-engineering/README.md)

**دمای مدل** - کنترل تصادفی بودن خروجی. پایین=قطعی، بالا=خلاقانه.

**پایگاه داده برداری** - پایگاه داده تخصصی برای تعابیر. [ماژول ۰۳](../03-rag/README.md)

**یادگیری صفر نمونه‌ای** - انجام وظایف بدون نمونه. [ماژول ۰۲](../02-prompt-engineering/README.md)

## محافظت‌ها - [ماژول ۰۰](../00-quick-start/README.md)

**دفاع در عمق** - رویکرد امنیتی چند لایه که حفاظ‌های سطح برنامه را با فیلترهای ایمنی ارائه دهنده ترکیب می‌کند.

**مسدودسازی سخت** - ارائه‌دهنده خطای HTTP 400 برای تخلفات شدید محتوا صادر می‌کند.

**InputGuardrail** - رابط LangChain4j برای تأیید ورودی کاربر قبل از رسیدن به LLM. صرفه‌جویی در هزینه و تأخیر با مسدود کردن پرامپت‌های مضر زودهنگام.

**InputGuardrailResult** - نوع برگشتی برای اعتبارسنجی محافظ: `success()` یا `fatal("دلیل")`.

**OutputGuardrail** - رابط برای تأیید پاسخ‌های AI قبل از بازگشت به کاربران.

**فیلترهای ایمنی ارائه‌دهنده** - فیلترهای محتوای داخلی از ارائه‌دهندگان AI (مثلاً GitHub Models) که تخلفات را در سطح API تشخیص می‌دهند.

**رد نرم** - مدل مودبانه از پاسخ خودداری می‌کند بدون اینکه خطا تولید کند.

## مهندسی پرامپت - [ماژول ۰۲](../02-prompt-engineering/README.md)

**زنجیره تفکر** - استدلال گام به گام برای دقت بهتر.

**خروجی محدود شده** - اعمال فرمت یا ساختار خاص.

**اشتیاق بالا** - الگوی GPT-5 برای استدلال کامل.

**اشتیاق پایین** - الگوی GPT-5 برای پاسخ‌های سریع.

**گفتگوی چند نوبتی** - حفظ زمینه در طول تبادل‌ها.

**پرامپت براساس نقش** - تنظیم پرسونا مدل از طریق پیام‌های سیستمی.

**خودبازتابی** - مدل خروجی خود را ارزیابی و بهبود می‌دهد.

**تحلیل ساختاریافته** - چارچوب ارزیابی ثابت.

**الگوی اجرای کار** - برنامه‌ریزی → اجرا → خلاصه‌سازی.

## RAG (تولید تکمیلی بازیابی شده) - [ماژول ۰۳](../03-rag/README.md)

**خط لوله پردازش سند** - بارگذاری → تقسیم → تعبیه → ذخیره.

**ذخیره تعبیه در حافظه** - ذخیره‌سازی غیرپایدار برای آزمایش.

**RAG** - ترکیب بازیابی با تولید برای استناد پاسخ‌ها.

**امتیاز شباهت** - اندازه‌گیری (۰-۱) از شباهت معنایی.

**ارجاع منبع** - فراداده درباره محتوای بازیابی شده.

## عامل‌ها و ابزارها - [ماژول ۰۴](../04-tools/README.md)

**حاشیه‌نویسی @Tool** - نشانه‌گذاری متدهای جاوا به عنوان ابزارهای قابل فراخوانی AI.

**الگوی ReAct** - استدلال → عمل → مشاهده → تکرار.

**مدیریت جلسه** - زمینه‌های جداگانه برای کاربران مختلف.

**ابزار** - تابعی که یک عامل AI می‌تواند فراخوانی کند.

**شرح ابزار** - مستندسازی هدف و پارامترهای ابزار.

## ماژول عاملی - [ماژول ۰۵](../05-mcp/README.md)

**حاشیه‌نویسی @Agent** - نشانه‌گذاری رابط‌ها به عنوان عامل‌های AI با تعریف رفتار اعلامی.

**شنونده عامل** - قلاب برای نظارت بر اجرای عامل از طریق `beforeAgentInvocation()` و `afterAgentInvocation()`.

**محدوده عاملی** - حافظه مشترک که در آن عامل‌ها خروجی‌ها را ذخیره می‌کنند با استفاده از `outputKey` برای استفاده عوامل پایین‌دستی.

**AgenticServices** - کارخانه برای ساخت عامل‌ها با استفاده از `agentBuilder()` و `supervisorBuilder()`.

**گردش کار شرطی** - مسیر یابی بر اساس شرایط به عوامل تخصصی مختلف.

**انسان در حلقه** - الگوی گردش کار با مشاهده‌گاه‌های انسانی برای تأیید یا بررسی محتوا.

**langchain4j-agentic** - وابستگی Maven برای ساخت عوامل اعلامی (تجربی).

**گردش کار حلقه‌ای** - اجرای تکراری عامل تا رسیدن به شرطی (مثلاً امتیاز کیفیت ≥ ۰.۸).

**outputKey** - پارامتر حاشیه‌نویسی عامل که مشخص می‌کند نتایج کجا در محدوده عاملی ذخیره شود.

**گردش کار موازی** - اجرای همزمان چند عامل برای وظایف مستقل.

**استراتژی پاسخ** - نحوه ارائه پاسخ نهایی توسط سرپرست: آخرین (LAST)، خلاصه (SUMMARY)، یا امتیازدهی شده (SCORED).

**گردش کار متوالی** - اجرای عامل‌ها به ترتیب که خروجی به مرحله بعدی منتقل می‌شود.

**الگوی عامل سرپرست** - الگوی عاملی پیشرفته که در آن LLM سرپرست به طور پویا تصمیم می‌گیرد کدام زیرعامل‌ها را فراخوانی کند.

## پروتکل زمینه مدل (MCP) - [ماژول ۰۵](../05-mcp/README.md)

**langchain4j-mcp** - وابستگی Maven برای ادغام MCP در LangChain4j.

**MCP** - پروتکل زمینه مدل: استانداردی برای اتصال برنامه‌های AI به ابزارهای خارجی. یکبار بساز، همه‌جا استفاده کن.

**کلاینت MCP** - برنامه‌ای که به سرورهای MCP متصل شده و ابزارها را کشف و استفاده می‌کند.

**سرور MCP** - سرویسی که ابزارها را از طریق MCP با شرح و ساختار پارامترهای شفاف عرضه می‌کند.

**McpToolProvider** - جز LangChain4j که ابزارهای MCP را برای استفاده در خدمات AI و عامل‌ها بسته‌بندی می‌کند.

**McpTransport** - رابط برای ارتباط MCP. پیاده‌سازی‌ها شامل Stdio و HTTP هستند.

**انتقال Stdio** - انتقال پردازش محلی از طریق stdin/stdout. برای دسترسی به سیستم فایل یا ابزارهای خط فرمان مفید است.

**StdioMcpTransport** - پیاده‌سازی LangChain4j که سرور MCP را به عنوان فرآیند فرعی آغاز می‌کند.

**کشف ابزار** - کلاینت از سرور برای ابزارهای موجود با شرح و ساختار پرس‌وجو می‌کند.
