# مشارکت

از علاقه‌مندی شما به مشارکت سپاسگزاریم! ما درخواست‌های پول و بهبودهای این دوره را خوش‌آمد می‌گوییم. لطفاً هنگام ارسال تغییرات این دستورالعمل‌ها را دنبال کنید:

1. **هیچ راز یا اطلاعات محرمانه‌ای در کامیت‌ها نباشد.** هرگز کلیدهای API یا هر ماده محرمانه‌ای را کامیت نکنید. برای تست‌های خود از یک فایل محلی `.env` استفاده کنید.
2. **تغییرات را متمرکز نگه دارید.** درخواست‌های پول کوچک و محدود آسان‌تر بررسی و نگهداری می‌شوند. اگر چندین مشکل را رفع می‌کنید، لطفاً آنها را جداگانه ارسال کنید.
3. **در صورت امکان تست اضافه کنید.** تست‌های واحد یا یکپارچه‌سازی به شناسایی بازگشت‌ها کمک می‌کنند. برای ماژول‌های Spring Boot، تست‌ها را در پوشه `src/test/java` قرار دهید.
4. **از قراردادهای سبک کد پیروی کنید.** از ساختار موجود پروژه به عنوان راهنما استفاده کنید. ما برای مدیریت وابستگی‌ها به Maven و برای بسته‌بندی برنامه‌ها به افزونه Spring Boot Maven تکیه می‌کنیم.

از اینکه به بهتر شدن این دوره کمک می‌کنید سپاسگزاریم!
