## பாதுகாப்பு

Microsoft எங்கள் மென்பொருள் தயாரிப்புகள் மற்றும் சேவைகளின் பாதுகாப்பை மிகுந்த முக்கியத்துவத்துடன் எடுத்துக்கொள்கிறது, இதில் எங்கள் GitHub நிறுவனங்கள் மூலம் நிர்வகிக்கப்படும் அனைத்து மூலக் குறியீட்டு சேமிப்பகங்களும் அடங்கும், அவை [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) மற்றும் [Xamarin](https://github.com/xamarin) ஆகியவற்றை உள்ளடக்கியவை.

நீங்கள் எந்த Microsoft-உரிமையுள்ள சேமிப்பகத்தில் [Microsoft-இன் பாதுகாப்பு குறைபாடு வரையறை](https://aka.ms/security.md/definition) பூர்த்தி செய்யும் பாதுகாப்பு குறைபாட்டை கண்டுபிடித்ததாக நினைத்தால், கீழே விவரிக்கப்பட்டபடி அதை எங்களுக்கு அறிவிக்கவும்.

## பாதுகாப்பு பிரச்சனைகள் அறிக்கை

**பொது GitHub பிரச்சனைகள் மூலம் பாதுகாப்பு குறைபாடுகளை அறிக்கையிட வேண்டாம்.**

பதிலாக, Microsoft பாதுகாப்பு பதிலளிப்பு மையம் (MSRC) இல் [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) என்ற முகவரியில் அவற்றை அறிக்கையிடவும்.

உள்நுழையாமல் சமர்ப்பிக்க விரும்பினால், [secure@microsoft.com](mailto:secure@microsoft.com) என்ற மின்னஞ்சலுக்கு அனுப்பவும். சாத்தியமானால், எங்கள் PGP விசையுடன் உங்கள் செய்தியை குறியாக்கவும்; அதை [Microsoft பாதுகாப்பு பதிலளிப்பு மையம் PGP விசை பக்கம்](https://aka.ms/security.md/msrc/pgp) இலிருந்து பதிவிறக்கம் செய்யவும்.

நீங்கள் 24 மணி நேரத்துக்குள் பதிலை பெறுவீர்கள். ஏதேனும் காரணத்தால் பெறவில்லை என்றால், உங்கள் அசல் செய்தி எங்களுக்கு வந்ததா என்பதை உறுதிப்படுத்த மின்னஞ்சல் மூலம் தொடரவும். கூடுதல் தகவல்கள் [microsoft.com/msrc](https://www.microsoft.com/msrc) இல் கிடைக்கின்றன.

தயவுசெய்து கீழே பட்டியலிடப்பட்ட தேவையான தகவல்களை (மிகவும் வழங்க முடிந்த அளவு) சேர்க்கவும், இது பிரச்சனையின் இயல்பு மற்றும் பரப்பை நன்றாக புரிந்துகொள்ள உதவும்:

  * பிரச்சனையின் வகை (எ.கா., பஃபர் ஓவர்ஃப்ளோ, SQL இன்ஜெக்ஷன், குறுக்குவலை ஸ்கிரிப்டிங், மற்றும் பிற)
  * பிரச்சனை வெளிப்படும் மூலக் கோப்புகளின் முழு பாதைகள்
  * பாதிக்கப்பட்ட மூலக் குறியீட்டின் இடம் (டேக்/பிராஞ்ச்/கமிட் அல்லது நேரடி URL)
  * பிரச்சனையை மீண்டும் உருவாக்க தேவையான சிறப்பு கட்டமைப்பு
  * பிரச்சனையை மீண்டும் உருவாக்க படி படி வழிமுறைகள்
  * சான்று-கான்செப்ட் அல்லது எக்ஸ்ப்ளாயிட் குறியீடு (சாத்தியமானால்)
  * பிரச்சனையின் தாக்கம், அதில் தாக்குதலாளி எப்படி பிரச்சனையை பயன்படுத்தலாம் என்பதைக் கொண்டுள்ளது

இந்த தகவல் உங்கள் அறிக்கையை விரைவாக பரிசீலிக்க உதவும்.

நீங்கள் பக் பவுண்டி அறிக்கையிடுகிறீர்களானால், முழுமையான அறிக்கைகள் அதிக பவுண்டி விருதுக்கு உதவும். எங்கள் [Microsoft பக் பவுண்டி திட்டம்](https://aka.ms/security.md/msrc/bounty) பக்கத்தை எங்கள் செயல்பாட்டுத் திட்டங்கள் பற்றிய கூடுதல் விவரங்களுக்கு பார்வையிடவும்.

## விருப்ப மொழிகள்

எங்கள் அனைத்து தொடர்புகளும் ஆங்கிலத்தில் இருக்க வேண்டும் என்று விரும்புகிறோம்.

## கொள்கை

Microsoft [ஒத்துழைக்கப்பட்ட பாதுகாப்பு குறைபாடு வெளிப்படுத்தல்](https://aka.ms/security.md/cvd) கொள்கையை பின்பற்றுகிறது.
