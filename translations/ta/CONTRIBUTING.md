# Contributing

உங்கள் பங்களிப்புக்கு நன்றி! இந்த பாடத்திட்டத்திற்கு புல்
ரிக்வெஸ்ட்கள் மற்றும் மேம்பாடுகளை வரவேற்கிறோம். மாற்றங்களை சமர்ப்பிக்கும் போது
தயவுசெய்து இந்த வழிகாட்டுதல்களை பின்பற்றவும்:

1. **கமிட்களில் ரகசியங்கள் இருக்கக்கூடாது.** API விசைகள் அல்லது எந்தவொரு
   ரகசியமான பொருளையும் ஒருபோதும் கமிட் செய்ய வேண்டாம். உங்கள் சொந்த சோதனைக்காக உள்ளூர் `.env` ஐ பயன்படுத்தவும்.
2. **மாற்றங்களை கவனமாக வைத்திருங்கள்.** சிறிய, குறுக்கப்பட்ட புல் ரிக்வெஸ்ட்கள்
   மதிப்பாய்வு மற்றும் பராமரிப்புக்கு எளிதாக இருக்கும். நீங்கள் பல பிரச்சனைகளை சரி செய்யும் போது,
   அவற்றை தனித்தனியாக சமர்ப்பிக்கவும்.
3. **சாத்தியமான இடங்களில் சோதனைகள் சேர்க்கவும்.** யூனிட் அல்லது ஒருங்கிணைப்பு சோதனைகள்
   மீள்பார்வைகளை பிடிக்க உதவும். Spring Boot மாட்யூல்களுக்கு, `src/test/java` கோப்புறையின் கீழ் சோதனைகளை சேர்க்கவும்.
4. **கோடு பாணி வழிமுறைகளை பின்பற்றவும்.** உள்ளமைவான திட்ட அமைப்பை வழிகாட்டியாக பயன்படுத்தவும்.
   நாங்கள் Maven ஐ சார்புகளை நிர்வகிக்கவும் மற்றும் Spring Boot Maven பிளகினை பயன்பாடுகளை தொகுப்பதற்கும் நம்புகிறோம்.

இந்த பாடத்திட்டத்தை மேம்படுத்த உதவியதற்கு நன்றி!
