# المساهمة

شكرًا لاهتمامك بالمساهمة! نحن نرحب بطلبات السحب والتحسينات على هذه الدورة. يرجى اتباع هذه الإرشادات عند تقديم التغييرات:

1. **لا أسرار في الالتزامات.** لا تقم أبدًا بالتزام مفاتيح API أو أي مواد سرية. استخدم ملف `.env` محليًا للاختبار الخاص بك.
2. **حافظ على تركيز التغييرات.** طلبات السحب الصغيرة والمحددة أسهل في المراجعة والصيانة. إذا كنت تصلح مشكلات متعددة، يرجى تقديمها بشكل منفصل.
3. **أضف اختبارات حيثما أمكن.** تساعد اختبارات الوحدة أو التكامل في اكتشاف التراجعات. بالنسبة لوحدات Spring Boot، قم بتضمين الاختبارات تحت مجلد `src/test/java`.
4. **اتبع قواعد نمط الكود.** استخدم هيكل المشروع الحالي كدليل. نعتمد على Maven لإدارة التبعيات وعلى مكون Spring Boot Maven لتجميع التطبيقات.

شكرًا لمساعدتك في تحسين هذه الدورة!
