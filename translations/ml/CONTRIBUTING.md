# സംഭാവന

സംഭാവനയ്ക്ക് താൽപര്യം കാണിച്ചതിന് നന്ദി! ഈ കോഴ്സിലേക്ക് പുൾ
റിക്വസ്റ്റുകളും മെച്ചപ്പെടുത്തലുകളും സ്വാഗതം ചെയ്യുന്നു. മാറ്റങ്ങൾ സമർപ്പിക്കുമ്പോൾ ദയവായി ഈ
നിർദ്ദേശങ്ങൾ പാലിക്കുക:

1. **കമ്മിറ്റുകളിൽ രഹസ്യങ്ങൾ ഇല്ല.** API കീകൾ അല്ലെങ്കിൽ ഏതെങ്കിലും
   രഹസ്യമായ വസ്തുക്കൾ ഒരിക്കലും കമ്മിറ്റ് ചെയ്യരുത്. നിങ്ങളുടെ സ്വന്തം ടെസ്റ്റിംഗിനായി ഒരു ലോക്കൽ `.env` ഉപയോഗിക്കുക.
2. **മാറ്റങ്ങൾ കേന്ദ്രീകരിക്കുക.** ചെറിയ, പരിധിയുള്ള പുൾ റിക്വസ്റ്റുകൾ അവലോകനം ചെയ്യാനും പരിപാലിക്കാനും എളുപ്പമാണ്. നിങ്ങൾ പല പ്രശ്നങ്ങളും പരിഹരിക്കുന്നുവെങ്കിൽ, ദയവായി അവ വേർതിരിച്ച് സമർപ്പിക്കുക.
3. **സാധ്യമായിടത്ത് ടെസ്റ്റുകൾ ചേർക്കുക.** യൂണിറ്റ് അല്ലെങ്കിൽ ഇന്റഗ്രേഷൻ ടെസ്റ്റുകൾ പുനരാവർത്തനങ്ങൾ പിടികൂടാൻ സഹായിക്കുന്നു. സ്പ്രിംഗ് ബൂട്ട് മോഡ്യൂളുകൾക്കായി, `src/test/java` ഫോൾഡറിനുള്ളിൽ ടെസ്റ്റുകൾ ഉൾപ്പെടുത്തുക.
4. **കോഡ് സ്റ്റൈൽ പരമ്പരാഗതങ്ങൾ പാലിക്കുക.** നിലവിലുള്ള പ്രോജക്ട് ഘടനയെ മാർഗ്ഗനിർദ്ദേശമായി ഉപയോഗിക്കുക. ആശ്രിതങ്ങൾ നിയന്ത്രിക്കാൻ Maven-നെ ആശ്രയിക്കുന്നു, ആപ്ലിക്കേഷനുകൾ പാക്കേജ് ചെയ്യാൻ സ്പ്രിംഗ് ബൂട്ട് Maven പ്ലഗിൻ ഉപയോഗിക്കുന്നു.

ഈ കോഴ്സ് മെച്ചപ്പെടുത്താൻ സഹായിച്ചതിന് നന്ദി!
