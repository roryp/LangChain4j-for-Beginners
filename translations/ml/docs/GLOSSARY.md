## പ്രധാന ആശയങ്ങൾ

**AI Agent** - സ്വയം നിര്ണയം ചെലുത്തുകയും സ്വതന്ത്രമായി പ്രവർത്തിക്കുകയും ചെയ്യാൻ AI ഉപയോഗിക്കുന്ന സിസ്റ്റം. [മൊഡ്യൂൾ 04](../04-tools/README.md)

**ചെയിൻ** - പ്രവർത്തനങ്ങളുടെ ഒരു പരമ്പര; ഔട്ട്‌പുട്ട് അടുത്ത ഘട്ടത്തിലേക്കാണ് നൽകപ്പെടുന്നത്.

**Chunking** - രേഖകളെ ചെറിയ ഭാഗങ്ങളായി വിഭജിക്കൽ. സാധാരണ: 300-500 ടോക്കണുകൾ ഓവർലാപ്പോടെ. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**Context Window** - മോഡൽ പ്രോസസ് ചെയ്യാവുന്ന പരമാവധി ടോക്കണുകൾ. GPT-5: 400K ടോക്കൺസ്.

**Embeddings** - ടെക്സ്റ്റിന്റെ അർഥം പ്രതിനിധാനം ചെയ്യുന്ന സംഖ്യാ വെക്റ്ററുകൾ. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**Function Calling** - മോഡൽ പുറം ഫംഗ്ഷനുകൾ വിളിക്കാനുള്ള ഘടനാപരമായ അഭ്യർത്ഥനകൾ ജനറേറ്റ് ചെയ്യുക. [മൊഡ്യൂൾ 04](../04-tools/README.md)

**Hallucination** - മോഡലുകൾ തെറ്റായ പക്ഷേ സാദ്ധ്യമായ വിവരങ്ങൾ സൃഷ്ടിക്കുമ്പോൾ.

**Prompt** - ഒരു ഭാഷാ മോഡലിന് നൽകിയ ടെക്സ്റ്റ് ഇൻപുട്ട്. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Semantic Search** - എംബഡിംഗുകൾ ഉപയോഗിച്ച് അർത്ഥത്തെ അടിസ്ഥാനമാക്കി തിരയൽ, കീവർഡുകൾ ഉപയോഗിക്കാതെ. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: മെമ്മറി ഇല്ല. Stateful: സംഭാഷണ ചരിത്രം നിലനിർത്തുന്നു. [മൊഡ്യൂൾ 01](../01-introduction/README.md)

**Tokens** - മോഡലുകൾ പ്രോസസ് ചെയ്യുന്ന അടിസ്ഥാന ടെക്സ്റ്റ് യൂണിറ്റുകൾ. ചെലവുകളും പരിധികളെയും ബാധിക്കുന്നു. [മൊഡ്യൂൾ 01](../01-introduction/README.md)

**Tool Chaining** - ടൂളുകൾ പരമ്പരയായി നടപ്പിലാക്കൽ; ഔട്ട്പുട്ട് അടുത്ത കോൾക്ക് വിവരം നൽകുന്നു. [മൊഡ്യൂൾ 04](../04-tools/README.md)

## LangChain4j ഘടകങ്ങൾ

**AiServices** - ടൈപ്പ്-സേഫ് AI സേവന ഇന്റർഫേസുകൾ സൃഷ്ടിക്കുന്നു.

**OpenAiOfficialChatModel** - OpenAI болон Azure OpenAI മോഡലുകൾക്കുള്ള സംയോജിത ക്ലയന്റ്.

**OpenAiOfficialEmbeddingModel** - OpenAI Official ക്ലയന്റ് ഉപയോഗിച്ച് എംബഡിംഗുകൾ സൃഷ്ടിക്കുന്നു (OpenAIയും Azure OpenAIയും പിന്തുണയ്ക്കുന്നു).

**ChatModel** - ഭാഷാ മോഡലുകളുടെ കോർ ഇന്റർഫേസ്.

**ChatMemory** - സംഭാഷണ ചരിത്രം നിലനിർത്തുന്നു.

**ContentRetriever** - RAG-യ്ക്ക് അനുയോജ്യമായ രേഖാ ഭാഗങ്ങൾ കണ്ടെത്തുന്നു.

**DocumentSplitter** - രേഖകൾ ചങ്കുകളായി വിഭജിക്കുന്നു.

**EmbeddingModel** - ടെക്സ്റ്റ് സംഖ്യാത്മക വെക്റ്ററുകളായി മാറ്റുന്നു.

**EmbeddingStore** - എംബഡിംഗുകൾ സംഭരിക്കുകയും വീണ്ടെടുക്കുകയും ചെയ്യുന്നു.

**MessageWindowChatMemory** - പുതിയ സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ് വിൻഡോ നിലനിർത്തുന്നു.

**PromptTemplate** - `{{variable}}` പ്ലേസ്‌ഹോൾഡറുകൾ ഉപയോഗിച്ച് പുനരുപയോഗയോഗ്യമായ പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുന്നു.

**TextSegment** - മെറ്റാഡേറ്റയോടെ ഉള്ള ടെക്സ്റ്റ് ചങ്ക്. RAG-ൽ ഉപയോഗിക്കുന്നു.

**ToolExecutionRequest** - ടൂൾ എക്സിക്യൂഷൻ അഭ്യർത്ഥനയെ പ്രതിനിധീകരിക്കുന്നു.

**UserMessage / AiMessage / SystemMessage** - സംഭാഷണ സന്ദേശ തരംകൾ.

## AI/ML ആശയങ്ങൾ

**Few-Shot Learning** - പ്രോംപ്റ്റുകളിൽ ഉദാഹരണങ്ങൾ നൽകൽ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - വിപുലമായ ടെക്സ്റ്റ് ഡാറ്റയിൽ പരിശീലിപ്പിച്ച AI മോഡലുകൾ.

**Reasoning Effort** - ചിന്തയുടെ ആഴം നിയന്ത്രിക്കുന്ന GPT-5 പാരാമീറ്റർ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Temperature** - ഔട്ട്പുട്ടിന്റെ പ്രത്യാശാസ്യത നിയന്ത്രിക്കുന്നു. താഴ്ന്നത് = നിശ്ചിതം, ഉയർന്നത് = സൃഷ്ടിശീലമാകുന്നു.

**Vector Database** - എംബഡിംഗുകൾക്കുള്ള പ്രത്യേക ഡേറ്റാബേസ്. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**Zero-Shot Learning** - ഉദാഹരണങ്ങളില്ലാതെ ജോലികൾ നിർവഹിക്കൽ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

## പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് - [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Chain-of-Thought** - മെച്ചപ്പെട്ട കൃത്യതയ്ക്കായി ഘട്ടം-ഘട്ടമായ ചിന്തനാപദ്ധതി.

**Constrained Output** - നിശ്ചിത ഫോർമാറ്റ് അല്ലെങ്കിൽ ഘടന പാലിച്ചുള്ള ഔട്ട്പുട്ട് നിര്‍ബന്ധിതമാക്കല്‍.

**High Eagerness** - സമഗ്രമായ ചിന്തനയ്ക്കുള്ള GPT-5 പാറ്റേൺ.

**Low Eagerness** - വേഗത്തിൽ മറുപടി നൽകുന്നതിനുള്ള GPT-5 പാറ്റേൺ.

**Multi-Turn Conversation** - എക്സ്ചേഞ്ചുകൾക്കിടയിൽ കോൺടെക്സ്റ്റ് നിലനിർത്തൽ.

**Role-Based Prompting** - സിസ്റ്റം സന്ദേശങ്ങളിലൂടെ മോഡലിന്റെ വ്യക്തിത്വം നിശ്ചയിക്കൽ.

**Self-Reflection** - മോഡൽ തന്റെ ഔട്ട്പുട്ട് വിലയിരുത്തുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യുന്നു.

**Structured Analysis** - നിശ്ചിതമായ മൂല്യനിർണയ ഫ്രെയിംവർക്ക്.

**Task Execution Pattern** - പദ്ധതി → നടപ്പാക്കുക → സംക്ഷേപിക്കുക.

## RAG (റെട്രീവൽ-ഓഗ്മെന്റഡ് ജെനറേഷൻ) - [മൊഡ്യൂൾ 03](../03-rag/README.md)

**Document Processing Pipeline** - Load → chunk → embed → store.

**In-Memory Embedding Store** - ടെസ്റ്റിംഗിനുള്ള സ്ഥിരതയില്ലാത്ത സംഭരണം.

**RAG** - റിട്രീവലും ജനറേഷനും സംയോജിപ്പിച്ച് മറുപടികളെ ഭൂമികാമാക്കുന്നു.

**Similarity Score** - അർഥപരമായ സമാനതയുടെ (0-1) അളവ്.

**Source Reference** - കണ്ടെത്തിയ ഉള്ളടക്കത്തെക്കുറിച്ചുള്ള മെറ്റാഡേറ്റ.

## ഏജന്റുകളും ടൂളുകളും - [മൊഡ്യൂൾ 04](../04-tools/README.md)

**@Tool Annotation** - Java മെത്തഡുകളെ AI വിളിക്കാവുന്ന ടൂളുകളായി അടയാളപ്പെടുത്തുന്നു.

**ReAct Pattern** - ന്യായീകരിക്കുക → പ്രവർത്തിക്കുക → നിരീക്ഷിക്കുക → ആവർത്തിക്കുക.

**Session Management** - വ്യത്യസ്ത ഉപയോക്താക്കൾക്കായുള്ള വേർതിരിഞ്ഞ കോൺടെക്സ്റ്റുകൾ.

**Tool** - AI ഏജന്റ് വിളിക്കാൻ കഴിയുന്ന ഒരു ഫംഗ്ഷൻ.

**Tool Description** - ടൂളിന്റെ ഉദ്ദേശവും പാരാമീറ്ററുകളും സംബന്ധിച്ച ഡോക്യുമെന്റേഷൻ.

## മോഡൽ കോൺടെക്‌സ് പ്രോട്ടോക്കോൾ (MCP) - [മൊഡ്യൂൾ 05](../05-mcp/README.md)

**MCP** - AI ആപ്പുകളെ ബാഹ്യ ടൂളുകളുമായി ബന്ധിപ്പിക്കുന്നതിന് ഒരു സ്റ്റാൻഡേർഡ്.

**MCP Client** - MCP സർവറുകളുമായി ബന്ധിപ്പിക്കുന്ന ആപ്ലിക്കേഷൻ.

**MCP Server** - MCP മുഖേന ടൂളുകൾ എക്സ്പോസ് ചെയ്യുന്ന സർവീസ്.

**Stdio Transport** - stdin/stdout വഴി subprocess ആയി സർവറിനെ പ്രവർത്തിപ്പിക്കൽ.

**Tool Discovery** - ലഭ്യമായ ടൂളുകൾ കണ്ടെത്താൻ ക്ലയന്റ് സർവറിനെ ചോദ്യം ചെയ്യുക.

## അസ്യൂർ സേവനങ്ങൾ - [മൊഡ്യൂൾ 01](../01-introduction/README.md)

**GPT-5** - ചിന്തന നിയന്ത്രണമുള്ള ഏറ്റവും പുതിയ OpenAI മോഡൽ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)
