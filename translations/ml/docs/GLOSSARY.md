## പ്രധാന ആശയങ്ങൾ

**AI ഏജന്റ്** - AI ഉപയോഗിച്ച് സ്വയം ചിന്തിക്കുകയും പ്രവർത്തിക്കുകയും ചെയ്യുന്ന സിസ്റ്റം. [മൊഡ്യൂൾ 04](../04-tools/README.md)

**ചെയിൻ** - ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക് അടിയറിഞ്ഞ പ്രവർത്തനങ്ങൾ തുടർച്ച.

**Chunking** - ഡോക്യുമെന്റുകൾ ചെറിയ ഭാഗങ്ങളാക്കി വിഭജിക്കൽ. സാധാരണ: 300-500 ടോക്കണുകൾ ഓവർലാപ്പുമായി. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**കോൺടെക്‌സ്റ്റ് വിൻഡോ** - മോഡൽ പ്രവർത്തിപ്പിക്കാൻ കഴിയുന്ന പരമാവധി ടോക്കണുകൾ. GPT-5: 400K ടോക്കൺസുകൾ.

**എംബെഡ്ഡിംഗ്സ്** - വാചകാർത്ഥത്തെ പ്രതിനിധീകരിക്കുന്ന സംഖ്യാത്മക വെക്ടറുകൾ. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**ഫംഗ്ഷൻ കോളിംഗ്** - മോഡൽ പുറത്തുള്ള ഫംഗ്ഷനുകൾ വിളിക്കാൻ ഘടനയുയർന്ന അഭ്യർത്ഥനകൾ സൃഷ്ടിക്കുന്നു. [മൊഡ്യൂൾ 04](../04-tools/README.md)

**ഹാല്യൂസിനേഷൻ** - മോഡലുകൾ തെറ്റായ പക്ഷേ വിശ്വസനീയമായ വിവരങ്ങൾ സൃഷ്ടിക്കുമ്പോൾ.

**പ്രോംപ്റ്റ്** - ഭാഷാ മോഡലിന് ചേർക്കുന്ന ടെക്സ്റ്റ്. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**സെമാന്റിക് സെർച്ച്** - കാര്യരൂപം ഉപയോഗിച്ചു തിരയൽ, കീവേഡുകൾക്കുപകരം എംബെഡ്ഡിംഗ്സുകൾ ഉപയോഗിച്ച്. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**സ്റ്റേറ്റ്‌ഫുൾ vs സ്റ്റേറ്റ്‌ലെസ്സ്** - സ്റ്റേറ്റ്‌ലെസ്സ്: ഓർമ്മയില്ല. സ്റ്റേറ്റ്‌ഫുൾ: സംഭാഷണ ചരിത്രം സൂക്ഷിക്കുന്നു. [മൊഡ്യൂൾ 01](../01-introduction/README.md)

**ടോക്കണുകൾ** - മോഡലുകൾ പ്രവർത്തിപ്പിക്കുന്ന അടിസ്ഥാന പാഠഘടകങ്ങൾ. ചെലവും പരിധികളിൽ സ്വാധീനം ചെലുത്തുന്നു. [മൊഡ്യൂൾ 01](../01-introduction/README.md)

**ടൂൾ ചെയിനിംഗ്** - തുടർച്ചയായ ടൂൾ പ്രവർത്തനം, ഔട്ട്പുട്ട് അടുത്ത വിളിപ്പാടിനായി പ്രഖ്യാപിക്കുന്നു. [മൊഡ്യൂൾ 04](../04-tools/README.md)

## LangChain4j ഘടകങ്ങൾ

**AiServices** - ടൈപ്പ്-സേഫ് AI സർവീസ് ഇന്റർഫേസുകൾ സൃഷ്ടിക്കുന്നു.

**OpenAiOfficialChatModel** - OpenAIയും അസ്യൂർ OpenAI മോഡലുകളും ഒരുീകൃതമായ ക്ലയന്റ്.

**OpenAiOfficialEmbeddingModel** - OpenAI ഔഫിഷ്യൽ ക്ലയന്റ് ഉപയോഗിച്ച് എംബെഡ്ഡിംഗ്സ് സൃഷ്ടിക്കുന്നു (OpenAIക്കും അസ്യൂർ OpenAIക്കും പിന്തുണയേകുന്നു).

**ChatModel** - ഭാഷാ മോഡലുകളുടെ പ്രധാന ഇന്റർഫേസ്.

**ChatMemory** - സംഭാഷണ ചരിത്രം സൂക്ഷിക്കുന്നു.

**ContentRetriever** - RAGയ്ക്കായി പ്രസക്തമായ ഡോക്യുമെന്റ് ചങ്കുകൾ കണ്ടെത്തുന്നു.

**DocumentSplitter** - ഡോക്യുമെന്റുകളെ ചങ്കുകളായി വിഭജിക്കുന്നു.

**EmbeddingModel** - ടെക്സ്റ്റ് സംഖ്യാത്മക വെക്ടറുകളാക്കി മാറ്റുന്നു.

**EmbeddingStore** - എംബെഡ്ഡിംഗുകൾ സൂക്ഷിക്കുകയും ലഭ്യവും.

**MessageWindowChatMemory** - പുതിയ സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ് വിൻഡോ സംരക്ഷിക്കുന്നു.

**PromptTemplate** - `{{variable}}` പ്ലേസ്‌ഹോൾഡറുകൾ ഉള്ള പുനഃഉപയോഗ സാധ്യമായ പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുന്നു.

**TextSegment** - മെറ്റഡാറ്റയുള്ള ടെക്സ്റ്റ് ചങ്ക്. RAGയിൽ ഉപയോഗിച്ചു പോകുന്നു.

**ToolExecutionRequest** - ടൂൾ പ്രവർത്തന അഭ്യർത്ഥനയെ പ്രതിനിധീകരിക്കുന്നു.

**UserMessage / AiMessage / SystemMessage** - സംഭാഷണ സന്ദേശ തരം.

## AI/ML ആശയങ്ങൾ

**Few-Shot Learning** - പ്രോംപ്റ്റുകളിൽ ഉദാഹരണങ്ങൾ നൽകൽ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**ലാർജ് ലാംഗ്വേജ് മോഡൽ (LLM)** - വിശാലമായ ടെക്സ്റ്റ് ഡാറ്റയിൽ പരിശീലനം ലഭിച്ച AI മോഡലുകൾ.

**റീസണിംഗ് എഫോർട്ട്** - ചിന്തന ആഴം നിയന്ത്രിക്കുന്ന GPT-5 പാരാമീറ്റർ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**ടെംപ്രേച്ചർ** - ഔട്ട്പുട്ടിന്റെ യാദৃച്ഛികത നിയന്ത്രിക്കുന്നു. കുറവുള്ളത്=നിശ്ചിതമായ, ഉയർന്നത്=സൃഷ്ടിപരമായ.

**വെക്ടർ ഡേറ്റാബേസ്** - എംബെഡ്ഡിംഗുകൾക്കായി പ്രത്യേക ഡേറ്റാബേസ്. [മൊഡ്യൂൾ 03](../03-rag/README.md)

**സീറോ-ഷോട്ട് ലേണിംഗ്** - ഉദാഹരണങ്ങൾ ഇല്ലാതെ പ്രവർത്തനങ്ങൾ ചെയ്യൽ. [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

## ഗാർഡ്രെയിൽസ് - [മൊഡ്യൂൾ 00](../00-quick-start/README.md)

**ഡിഫൻസ് ഇൻ ഡെപ്ത്** - ആപ്ലിക്കേഷൻ തല ഗാർഡ്രെയിൽസും പ്രൊവൈഡർ സേഫ്ടി ഫിൽറ്ററുകളും ചേർന്നുള്ള ബഹു-പതിവുഴക്ക സുരക്ഷ.

**ഹാർഡ് ബ്ലോക്ക്** - ഗൗരവമായ ഉള്ളടക്ക ലംഘനങ്ങളിൽ പ്രൊവൈഡർ HTTP 400 പിശക് നല്‍കുന്നു.

**InputGuardrail** - LLM-ൽ എത്തുക മുൻപ് ഉപയോക്തൃ ഇൻപുട്ട് പരിശോധിക്കാൻ LangChain4j ഇന്റർഫേസ്. നഷ്ടം കുറഞ്ഞും വൈകിയുമുള്ളതും കൃത്യമായി തടയുന്നു.

**InputGuardrailResult** - ഗാർഡ്രെയിൽ പരിശോധനയ്ക്ക് റിട്ടേൺ തരം: `success()` അല്ലെങ്കിൽ `fatal("കാരണവും")`.

**OutputGuardrail** - ഉപയോക്താക്കൾക്ക് നൽകുന്നതിന് മുമ്പ് AI മറുപടികൾ പരിശോധിക്കുന്ന ഇന്റർഫേസ്.

**Provider Safety Filters** - AI പ്രൊവൈഡർമാർ നൽകുന്ന ഉള്ളടക്ക ഫിൽറ്ററുകൾ (ഉദാ. GitHub മോഡലുകൾ) API തലത്തിൽ ലംഘനങ്ങൾ തടയുന്നു.

**Soft Refusal** - മോഡൽ പിശക് തീർക്കാതെ മാർഗ്ഗാനുബന്ധിച്ച് മറുപടി നൽകാത്തത്.

## പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് - [മൊഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**ചെയിൻ-ഓഫ്-തോട്ട്സ്** - കൂടുതൽ കൃത്യതക്കായി ഘട്ടം ഘട്ടമായ ചിന്തനം.

**കൺസ്ട്രെയ്നഡ് ഔട്ട്പുട്ട്** - നിശ്ചിത ഫോർമാറ്റ് അല്ലെങ്കിൽ ഘടന പ്രാവർത്തി ചെയ്യൽ.

**ഹൈ ഈഗർനെസ്** - സൂക്ഷ്മമായ ചിന്തനത്തിന് GPT-5 മാതൃക.

**ലോ ഈഗർനെസ്** - ദ്രുത പ്രതികരണത്തിന് GPT-5 മാതൃക.

**മൾട്ടി-ടേൺ സംഭാഷണം** - ഇടപാടുകൾക്കിടയിൽ കോൺടെക്‌സ്റ്റ് സൂക്ഷിക്കൽ.

**റോൾ-ബേസഡ് പ്രോംപ്റ്റിംഗ്** - മോഡലിന്റെ വ്യക്തിത്വം സിസ്റ്റം സന്ദേശങ്ങളിലൂടെ നിര്‍ണ്ണയം.

**സെൽഫ്-റിഫ്ലെക്ഷൻ** - മോഡൽ താൻ സൃഷ്ടിച്ച ഔട്ട്പുട്ട് വിലയിരുത്തുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യുന്നു.

**സ്റ്റ്രക്ചേഡ് അനാലിസിസ്** - നിശ്ചിതമായ വിലയിരുത്തൽ ഫ്രെയിംവർക്ക്.

**ടാസ്‌ക് എക്സിക്യൂഷൻ പാറ്റേൺ** - പദ്ധതി → നടപ്പാക്കൽ → സംഗ്രഹം.

## RAG (രിട്രീവൽ-ഓഗ്മെൻറഡ് ജനറേഷൻ) - [മൊഡ്യൂൾ 03](../03-rag/README.md)

**ഡോക്യുമെന്റ് പ്രോസസ്സ് പൈപ്പ്‌ലൈൻ** - ലോഡ് → ചങ്ക് → എംബഡ് → സ്റ്റോർ.

**ഇൻ-മെമ്മറി എംബഡിംഗ് സ്റ്റോർ** - ടെസ്റ്റിംഗിനുള്ള താൽക്കാലിക സംഭരണം.

**RAG** - മറുപടികൾക്കായി റിട്രീവലും ജനറേഷനും സംയോജിപ്പിക്കുന്നു.

**സിംilarity സ്കോർ** - സെമാന്റിക് സാമ്യമാണുള്ളത് 0-1 വരെ അളവുക.

**സോഴ്‌സ് റഫറൻസ്** - കണ്ടെത്തിയ ഉള്ളടക്കത്തിന്റെ മെറ്റഡാറ്റ.

## ഏജന്റുകളും ഉപകരണങ്ങളും - [മൊഡ്യൂൾ 04](../04-tools/README.md)

**@Tool അനോട്ടേഷൻ** - ജാവാ മെതഡുകൾ AI-കോളബിൾ ടൂളുകളായി അടയാളപ്പെടുത്തുന്നു.

**ReAct മാതൃക** - ചിന്തിക്കുക → പ്രവർത്തിക്കുക → നിരീക്ഷിക്കുക → ആവർത്തിക്കുക.

**സെഷൻ മാനേജ്മെന്റ്** - വിയത്യസ്ത ഉപയോക്താക്കൾക്കായി വ്യത്യസ്ത കോൺടെക്‌സ്റ്റുകൾ.

**ടൂൾ** - AI ഏജന്റ് വിളിക്കാൻ കഴിയുന്ന ഫംഗ്ഷൻ.

**ടൂൾ വിവരണം** - ടൂളിന്റെ ഉദ്ദേശവും പാരാമീറ്ററുകളും പത്രികപ്പെടുത്തൽ.

## എജന്റിക് മോഡ്യൂൾ - [മൊഡ്യൂൾ 05](../05-mcp/README.md)

**@Agent അനോട്ടേഷൻ** - ഇന്റർഫേസുകൾ AI ഏജന്റുകളായി, പ്രസക്തമായ പെരുമാറ്റ വിശദീകരണത്തോടെ അടയാളപ്പെടുത്തൽ.

**എജന്റ് ലിസ്ണർ** - `beforeAgentInvocation()` , `afterAgentInvocation()` വഴി ഏജന്റ് നടത്തിപ്പ് നിരീക്ഷിക്കാൻ ഹുക്ക്.

**എജന്റിക് സ്കോപ്പ്** - ഏജന്റുകൾ ഔട്ട്പുട്ടുകൾ സംഭരിക്കുന്നത് പങ്കിടുന്ന ഓർമ്മ, താഴെത്തട എജന്റുകൾ ഉപയോക്താക്കുന്നതിനായി `outputKey` ഉപയോഗിക്കുന്നു.

**AgenticServices** - `agentBuilder()` , `supervisorBuilder()` ഉപയോഗിച്ച് ഏജന്റുകൾ സൃഷ്ടിക്കുന്ന ഫാക്ടറി.

**കണ്ടീഷണൽ വർക്ക്‌ഫ്ലോ** - വ്യത്യസ്ത സ്പെഷ്യലിസ്റ്റ് ഏജന്റുകളിലേക്ക് വ്യത്യസ്ത അവസ്ഥകളുപരിച്ഛേദനം.

**ഹ്യൂമൺ-ഇൻ-ദ-ലൂപ്പ്** - അംഗീകാരം അല്ലെങ്കിൽ ഉള്ളടക്കപരിശോധനയ്ക്കായി മനുഷ്യ ചെക്ക്പോയിന്റുകൾ ചേർക്കുന്ന വർക്ക്‌ഫ്ലോ മാതൃക.

**langchain4j-agentic** - പ്രസക്തമായ ഏജന്റ് നിർമാണത്തിനുള്ള Maven ആശ്രയം (പരീക്ഷണ).

**ലൂപ് വർക്ക്‌ഫ്ലോ** - ഗുണനിലവാര സ്കോർ ≥ 0.8 വരെ ഏജന്റ് നിർവഹണം ആവർത്തിക്കുക.

**outputKey** - ഏജന്റ് അനോട്ടേഷൻ പാരാമീറ്റർ, ഏജന്റിക് സ്കോപ്പിൽ ഫലം സൂക്ഷിക്കുന്ന സ്ഥലം.

**പാരലൽ വർക്ക്‌ഫ്ലോ** - സ്വതന്ത്ര ജോലികൾക്കായി ഒരաժամത്ത് വിവിധ ഏജന്റുകൾ പ്രവർത്തിപ്പിക്കുക.

**റസ്പോൺസ് സ്ട്രാറ്റജി** - മേൽനോട്ടക്കാരൻ അന്തിമ ഉത്തരം രൂപപ്പെടുത്തുന്നത് എങ്ങനെ: LAST, SUMMARY, അല്ലെങ്കിൽ SCORED.

**സീക്വൻഷ്യൽ വർക്ക്‌ഫ്ലോ** - ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക് ഒഴുകുന്ന ക്രമത്തിൽ ഏജന്റുകൾ പ്രവർത്തിപ്പിക്കൽ.

**സൂപ്പർവൈസർ ഏജന്റ് മാതൃക** - ഏജന്റ്‍മിക പാറ്റേൺ, മേൽനോട്ട LLM ചുരുക്കപ്പെട്ട ഏജന്റുകൾ തിരഞ്ഞെടുക്കുന്നു.

## മോഡൽ കോൺടെക്‌സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) - [മൊഡ്യൂൾ 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j-ൽ MCP സംയോജിപ്പിക്കാനുള്ള Maven ആശ്രയം.

**MCP** - മോഡൽ കോൺടെക്‌സ്റ്റ് പ്രോട്ടോക്കോൾ: AI ആപ്പുകൾ ബാഹ്യ ഉപകരണങ്ങൾക്ക് കണക്ട് ചെയ്യാൻ സ്റ്റാൻഡേർഡ്. ഒരിക്കൽ നിർമ്മിക്കുക, എല്ലായിടത്തും ഉപയോഗിക്കുക.

**MCP ക്ലയന്റ്** - MCP സർവറുകളെ കണക്ട് ചെയ്ത് ടൂളുകൾ കണ്ടെത്താനും ഉപയോഗിക്കാനും ഉള്ള അപ്ലിക്കേഷൻ.

**MCP സർവർ** - ടൂൾസ് വ്യക്തമാക്കിയ വിവരണങ്ങളോടും പാരാമീറ്റർ സ്കീമകളോടും MCP വഴി ലഭ്യമായ സേവനം.

**McpToolProvider** - MCP ടൂളുകൾ AI സർവീസുകൾക്കും ഏജന്റുകൾക്കും LangChain4j-ൽ മൂടിയുള്ള ഘടകം.

**McpTransport** - MCP ആശയവിനിമയത്തിനായി ഇന്റർഫേസ്. സ്ട്രിങ്കുകൾ ഉൾപ്പെടുന്നു Stdio, HTTP.

**Stdio ട്രാൻസ്പോർട്ട്** - stdin/stdout വഴി പ്രാദേശിക പ്രോസസ് ട്രാൻസ്പോർട്ട്. ഫയൽസിസ്റ്റം ആക്സസ് അല്ലെങ്കിൽ കമാൻഡ് ലൈൻ ടൂൾസിന് ഉപകാരപ്രദം.

**StdioMcpTransport** - LangChain4j നടപ്പാക്കൽ MCP സർവർ subprocess ആയി ആരംഭിക്കുന്നു.

**ടൂൾ കണ്ടെത്തൽ** - ക്ലയന്റ് സർവറിനോട് ലഭ്യമായ ടൂളുകൾ വിവരണങ്ങളോടും സ്കീമകളോടും ചേർന്ന് അഭ്യർത്ഥിക്കുന്നു.
